DMAP export

NOTE: DMAP is an independent piece of computer software which is NOT supplied with Recorder. To export to DMAP in Recorder you need to install the add-in that provides this functionality.

General Description

DMAP produces Distribution Maps and Coincidence Maps which are displayed in colour on-screen, with multiple zoom capability. The maps can be printed at high resolution in colour or black and white, or they can be exported as high resolution images to other software for incorporating in reports or publications. It has been widely used in the biological recording community to produce maps to illustrate publications.

DMAP can accept distribution data as comma-delimited text (CSV) files from databases and spreadsheets. The distribution data can consist of Grid References, a variety of other metric co-ordinates, or Latitude+Longitude. This facility generates the necessary files from Taxon or Biotope occurrences stored by Recorder so that DMAP can be used to generate high resolution maps.

Exporting observations to DMAP

  • Select the observations you wish to export.

EITHER

  • Select a Location, Name and Address, Survey, Survey event, Sample, Taxon observation or Biotope observation in the appropriate window.

  • Select Tools – Export data from the Main menu.

OR

  • Use the Report Wizard to select observations.

  • Choose [Export data] in the Filter Result window.

  • In the Data Export dialog, select 'DMAP Export' as the Export Type.

  • Select the Destination file name. The button gives access to a standard Windows Save file dialog. Make sure that the correct folder is specified (usually \Dmapw) and make sure that you are specifying a file name ending with ".dis", then click on [Save]. The file name can be a new file name or the name of an existing file (in which case you will be asked to confirm that you want to replace the existing file).

  • Once you have selected a Destination file, the [OK] button will become active. Clicking [OK] will display the following dialog:

  • This dialog allows you to use measurement data associated with observations (i.e. taxon measurements or biotope measurements) to provide information which DMAP can display as different sizes, colours or shapes of symbols on the map.

    • If you do not wish to assign symbols according to quantities then choose the [Exclude Quantities] button. This will produce a simple distribution map indicating presence in grid squares.

    • If you wish to assign symbols using measurement data:

      • Choose the type of measurement you wish to use to assign symbols.

      • Enter a value between 2 and 9 for the number of symbols you wish to assign.
      • Enter a minimum and maximum value. These should be numeric values, and Max should be greater than Min.
      • Choose the [Include Quantities] button.

  • The export process will proceed and a message will be displayed indicating that it has been completed successfully. The result will be two files (with extensions ".dis", containing the distribution data, and ".nam", containing the names of the taxa or biotopes which were exported).

  • In DMAP you should create or open a file group which includes these two files. See the documentation supplied with DMAP for details.

How symbols are assigned

If you choose to assign symbols using quantitative data:

  • Only observations which have numeric measurements of the type you chose will be included in the export file.

  • The range from Min to Max is divided by the Number of Ranges to obtain the cut-points.

  • Any observation with a value at or below the Min Value is assigned to the smallest symbol class, and any observation with a value at or above the Max Value is assigned to the largest symbol class.

  • If you specify 2 as the 'Number of Ranges', you will only be asked to specify a single 'Cut Off' value which will act as the separation point for the allocation of Symbol 1 or 2.

Example:

If you specify:

  • 9 as the 'Number of Ranges';

  • 10 as the 'Min. Value';
  • 50 as the 'Max Value';

the following Symbol assignments will be made:

  • All data values less than 10 will be assigned Quantity Symbol 1.

  • All data values greater than 50 will be assigned Quantity Symbol 9.
  • Intermediate data values will be evenly divided into the remaining 7 ranges and assigned the appropriate Symbol in the series 2 through to 8.
