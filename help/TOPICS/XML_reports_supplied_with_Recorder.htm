XML reports supplied with Recorder

Version 6.15 of Recorder comes with numerous XML reports. Selecting Reports - Run from the main menu displays a hierarchy of the reports that can be run using that facility - for full details see Running Reports. The top level of the hierarchy lists the different categories of reports. Clicking the button beside a category displays the sub-categories or the XML reports themselves. Below is a list of the main categories followed by other information and a list of the XML reports in version 6.15. Note that a category will only appear if reports belonging to it are present.

See XML reports for more information, including an introduction to writing XML reports.

Report categories

Redundant reports

This category is included simply to allow the deprecation of reports that are no longer required. Rather than deleting them from your system, upgrades usually re-categorise them into this folder and you can decide yourself whether to formally delete them from your system. Note that if all redundant reports are deleted this folder will no longer appear.

Geographic Area/Location Reports

  • Bounding Box Reports - these use a bounding box defined by the spatial reference of the SW and NE corner. Note that these reports convert the SW and NE spatial references to Lat/Long and use the Lat/Long of the records to select the data. The effect of this is that records close to the boundaries can be included or excluded from the report when they might not be if the selection were based on grid references. Grid Square reports produce more accurate results, but only include squares which are totally within the bounding box.

  • Circle Reports - reports based on a circle defined by the grid reference of the centre point and the diameter of the circle.

  • Location hierarchy reports - reports based on Locations that are defined in the Location hierarchy (rather than the Location Name attribute on Survey Events and Samples). These can usually also be run by right clicking a Location in the Location hierarchy and selecting Quick Report - Geographic Area/Location Reports etc.

  • Location Name Reports - reports based on Location Name from Survey Events and Samples.

  • Polygon Reports - reports based on polygons on the Map. These can usually also be run by right clicking a polygon on the map and selecting Quick Report etc.

  • Grid Square Reports - reports based on Grid Squares. These reports work on the spatial reference and only select records where the box defined by the sample spatial reference falls totally within the square (squares) selected. These are more accurate than bounding box reports.

Biotope restricted reports

  • Biotope Reports - reports on Habitat (Biotope) data.

System reports

  • XML examples - example template included for tutorials etc.

  • Information - a variety of summary reports of the database contents such as possible duplicates, the lists in the taxon dictionary used to enter observations, etc.

  • Statistics - database statistics such as number of observations per survey, number of observations per taxon group etc.

Measurement restricted reports

  • Measurement Reports - reports based on Taxon Occurrence Measurements (e.g. Abundance).

Taxon restricted reports

  • Rucksack Reports - reports which select data based on Rucksacks

  • Taxon Reports - these reports work on taxa selected either by name or from a dictionary

  • Taxon Group - summary reports for a particular taxon group

Record validation

  • Validation Reports - these reports validate records for exporting to the NBN Gateway (e.g. non-system supplied names and non UK and Irish geographic coordinates).

Designations

  • Designation Reports - reports which provide taxon designation (status) information.

Report names

Each system supplied report is a separate XML file, with an extension of .xml, stored in the \User Files\Reports folder within your Recorder 6 folder unless the location has been changed using Tools - Options - File Locations. In a standard standalone installation with Windows XP they will be in C:\Program Files\Recorder 6\User Files\Reports. The file names of the JNCC supplied ones start with JNCC followed by a unique reference which can be used to refer to the report, e.g. Ln5. This is made up of an abbreviation of the report type followed by a number. The name of the report, as it appears in the Reports - Run hierarchy, is stored in the file and is usually the file name without the "JNCC" providing a link from the report name to the file. Some of the system supplied reports were provided by Sussex Biological Record Centre.

Report parameters

Many reports have parameters which allow you to specify what you want included in the report. Amongst these are often a set of filters which correspond to the constraints that can be set via the [Additional Filters] button on the Layout Selection page in the Report Wizard. These offer the ability to include or exclude:

  • Confidential Data

  • Unchecked Data
  • Failed/Pending Verification Data
  • Zero Abundance Data

In addition, reports which select observations based on spatial references or bounding boxes often have the ability to include or exclude observations with a Spatial Reference Qualifier of 'Site Centroid'. Although not labelled as such this is displayed beside spatial references in the observation hierarchy and elsewhere.

Report output

When an XML report is run the results are displayed in the filter result, in the same spreadsheet format as the results from the report wizard. From the filter result you can choose what to do with the selected data by clicking on the [Report output] button and selecting one the options that appear, e.g. Copy All to copy all the data to the Windows clipboard so that it can be pasted into a file such as an excel spreadsheet or word document, Send to Map to do a distribution map of the data, Export to Excel to save the data in a new excel file, etc. Any options that aren't available for the type of report you have done will be greyed out.

When New Report Template is selected the report designer opens so that you can design a report layout which will enable you to print the data in a format suitable for giving to customers - see Report designer for details. Note that at present the Report Designer isn't always reliable.

Modifying reports

The system supplied reports will produce useful results but can also be adapted to meet your own needs. Alternatively, the techniques they include can be used to produce new reports.

You should not change the actual system supplied reports, but take a copy of the file and rename it for your own use. If you don't do this then, for example, if an upgrade contains an updated version of it, this may overwrite your changed version. It is suggested that you replace the JNCC in the file name with your Site ID to make a unique file name, e.g. JNCC_Sq2_GridSquare.xml would become SR000027_Sq2a_GridSquare.xml. See Report names above for the location of the files.

The XML reports are text files that can be opened by a number of programs but they will be easiest to edit in Notepad. Right click on the file and select "Open With" then "Notepad", if that option is available. If not, select "Choose Program" then select "Notepad".

To differentiate your report from the original, you should edit the menu path, title and description in the <CustomReport tag near the start of the file as this is the information that appears in the report hierarchy in Recorder to enable you to locate and run it. You should also edit the comment at the top, i.e. the information between <!-- and --> as recommended in the note in XML reports.

For example: Change:

<CustomReport menupath="Geographic Area/Location Reports\Grid Square Reports"
title="Sq2_Summary by Square(s)" description="Displays records for specified Grid Square ..." > to:

<CustomReport menupath="SR000027 Reports"
title="Sq2a_Summary by Square(s) for SR000027" description="Displays records for specified Grid Square ... Modified for SR000027" >

Amongst the easiest things to change in a report are the column widths, the order in which they appear and the column headings. In most reports the column layout is defined near the end of the report, so for example, to change the width of the spatial reference column in Sq2, change 60 to 80 in:

<Column name="Spatial_Ref" width="60" position="2" caption="Spatial Ref" />

To change the order in which the columns appear, simply change the position attributes for each column. Similarly, you can change the column heading by altering the caption attribute. If you don't want a column to appear in the report add a visible attribute to the column and set it to false, e.g. visible="false".

It is also relatively easy to add in extra columns if they are for data from tables that are already used, e.g. Spatial_Ref_Qualifier if the Sample table is already used. In Sq2 you would add "S.Spatial_Ref_Qualifier," after "S.Spatial_ref," in the SELECT clause near the start and an appropriate column name clause, such as <Column name="Spatial_ref_qualifier" width="80" position="3" caption="Spatial Ref Qual" />. Note that the positions of the other columns would need to be changed. Also, the S in "S.Spatial_ref," is an alias for the Sample table as defined in the FROM clause.

See XML reports for an introduction to writing XML reports.
