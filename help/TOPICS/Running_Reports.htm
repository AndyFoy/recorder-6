Running Reports

The way you run reports will depend on how they have been designed to be run. Two options involve selecting Reports from the main menu, then:

  • Run, to run pre-prepared reports.

  • Quick Report, to run standard pre-prepared reports for an item selected in one of the Recorder hierarchies.

These options enable you to run:

  • The Standard Reports supplied with Recorder.

  • Reports you have produced and saved using the Report Wizard, with or without a template (layout).
  • Snapshots you have produced and saved using the Report Wizard.
  • XML Reports.

XML reports supplied with Recorder

Recorder comes with an extensive selection of XML reports, a powerful addition to the reporting tools in the system. If you require a report that can't be produced using the other reporting facilities you may find that it is already available as an XML report. Alternatively, you will be able to develop it yourself, provided you have access to the necessary computing skills. If not, one of the Recorder resellers may be willing to develop it for you.

These reports can be run from within your Recorder system using one of the methods described in this topic. They can also be designed to run from a selected item in a Recorder window.

The XML reports supplied in version 6.14 fall into various categories:

  • Geographical area / location reports

  • Biotope restricted reports
  • System reports, including statistical reports
  • Measurement restricted reports
  • Taxon restricted reports
  • Record validation

For more information on these reports, see XML reports supplied with Recorder. For introductory information on developing XML reports please click here.

Note: Reports that have not been produced by JNCC should be prefixed with information on their origin, e.g. a SITE ID or an organisation name. Comments in each report should include the author, a version number and the date the report was produced and/or amended. If you amend a report supplied with Recorder, it should be renamed to make this clear to anyone who uses it. JNCC and the other report providers accept no responsibility for the reports supplied, and advise users to test the output of each report to ensure that it is accurate and appropriate for their needs.

To run XML reports use Reports - Quick Report or Reports - Run as outlined below.

Reports - Quick Report

The reports that can be run using Reports - Quick Report depend on what is currently selected in the main windows (Observations, Locations, Names and Addresses and Documents) or in the taxon or biotope dictionary. For example, if a location is selected, and no reports have been saved or added to Recorder, the Quick Report menu will look something like this:

Selecting:

  • Occurrences for Places Report will run that report.

  • Places for Occurrences Report will run that report.

  • Selecting one of the items in the right hand list will run that XML report.

The result will be a completed report in the Print Preview window, ready to be printed or saved. It will show you the current state of the data in your database.

Note: Other reports may be available on the Quick Report menu if you have added reports to your system.

Note: The Quick Report menu also becomes available if you right click on something appropriate in one of the Recorder hierarchies, e.g. a taxon in the taxon dictionary or a survey in the observation hierarchy.

Reports - Run

Reports - Run is another way of running reports. As on the Quick Report menu, the options available will depend on whether you have saved reports produced using the Report Wizard, or added reports to your system. M1 Records for Measurement and M2 Records for Abundance are examples of XML reports which can be run from this menu. The XML report file itself specifies its position in this hierarchy.

When you select a report in the hierarchy, a description of it will appear in the box on the right.

Report Directory

This defaults to the folder specified under 'Queries' in the Tools - Options - File locations window. You can, however, use reports saved in any folder to which you have access. The button opens a standard Windows folder browser dialog from which you can choose another folder.

Select report

The list shows files with a '.wzd' or '.xml' extension in the selected directory. If there is a snapshot or template associated with the report, this will be displayed in brackets.

Once you have selected a report, the [OK] button will become active. To run the report, click [OK] and follow the instructions, e.g. you may be asked to supply parameters. The result will be a completed report in the Print Preview window, ready to be printed or saved. It will show you the current state of the data in your database. Click on the [Report Output] button to list the options available for further actions. The [Cancel] button can be used to abort the process.

Note: You can change the default location of your reports using Tools - Options - File Locations.
