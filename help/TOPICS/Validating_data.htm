Validating data

When entering data into Recorder it is validated using a Standard Validation Library. Many checks are done such as checking that dates are valid vague dates, that spatial references are valid for the selected spatial reference system (Tools - Options - Spatial References) and that, taxa, biotopes, locations, individuals, organisations and documents exist in the the appropriate list or hierarchy. Checks are also done for consistency, e.g. that the dates of observations are within the allowed date range for the survey to which they are to be added, and that the spatial references are within the bounding box of the survey. However, there are circumstances in which data can become invalid. A common example of this is when editing dates for observations. If a survey event, its samples and its taxon determinations all have dates of, say, 12/05/2008 but then a user changes the date in the survey event to, say, 19/05/2008 without changing the dates in the samples, the samples become invalid because their dates are no longer equal to or within the survey event dates. If sample dates are changed but not the taxon determination dates of the observations in them, the taxon determinations will become invalid if their dates are no longer equal to or after their sample date. Similarly with location data, the sample spatial reference must be equal to or within the survey event location, i.e. within the grid squares of the survey event location.

Invalid items can easily go unnoticed but will cause problems when exchanging data between systems. When importing data, it is validated using the Standard Validation Library and invalid items are not imported - see Invalid Items for more information.

In 2008, new facilities were introduced to help prevent items from becoming invalid and to detect any existing ones:

Editing dates and location data

When editing dates and location data in survey events you have the option to cascade the change through the samples in the event.
When editing dates and location data in samples you have the option to change the survey event at the same time, and the other samples in the same event.
In the case of changing dates the change can also be cascaded through the determinations.

To use this facility, edit the item as normal and make the required changes. If a date, location, location name or spatial reference has been changed, a dialogue box like this will appear when you click [Save].

These options appear when editing the date in a survey event. Select your desired option:

  • Do Nothing - only does the change in the item you have selected for editing.

  • Update all the child samples and determinations to match the survey event's updated Date - changes the date in the survey event, all its samples and all their determinations.

  • Update the child sample and determinations to match the survey event's updated Date but only if the values previously matched between the sample and survey event or determination - changes the date in the survey event, its samples and determinations but it only changes dates in samples and determinations which were the same as the event date beforehand.

Click [OK] to update the database and make the changes permanent.

When editing sample data an additional option is available: Also update all samples and determinations within the same survey event using the same rules.

Note: the second and third messages will vary depending on whether you are editing a survey event or sample and on whether you are editing a date, location, location name and/or spatial reference.

Revalidating items

If at any time you are concerned that your system may contain invalid data you can revalidate all data in a selected item:

To revalidate a survey tag, i.e all surveys with a particular tag, a survey, survey event, sample, taxon or biotope occurrence:

  • Select the required item in the observation hierarchy.

  • Right click and select Revalidate Record from the pop-up menu that appears.

If the revalidation doesn't find any invalid items the system will respond saying 'The database validation was successful.'

If the revalidation finds invalid items the system will display a box that lists the invalid items with a message to say why the item has failed validation. It will also offer two options:

  • [Save Details] - allows you to save the details of the invalid items in a file that can be opened in Notepad, Word, Excel or similar software. It can also be used as a filter that can be loaded to enable you to view the invalid items at a later date as when you select [View Results].

  • [View Results] - places a filter on the window concerned so that the invalid items are on view when items are expanded by clicking on the beside them. The invalid items will be highlighted and a message saying what is wrong with them will appear when you float the cursor over them. To remove the filter click [Show All] to the bottom right of the window.

For details of how to load a saved filter and the format of a filter file, see Loading filters.

Note: other user added data, such as locations, can be revalidated in the same way.

Note: this facility is only available to System Managers - see User access levels.

Revalidating data selected by the Report Wizard

To revalidate the observations in the report wizard's filter result go through the steps involved in using the wizard as normal until you are at the filter result. At this point:

  • Click the [Report Output] button.

  • Select Revalidate Items.

If the revalidation doesn't find any invalid items the system will respond saying 'The database validation was successful.'

If the revalidation finds invalid items the system will display a box that lists the invalid items with a message to say why the item has failed validation. It will also offer the two options above: [Save Details] and [View Results].

Note: this facility is only available to System Managers - see User access levels.

Revalidating the entire database

To revalidate the entire database:

  • Select Tools - Database Tools - Revalidate Database from the Main menu.

  • You will then be warned that this may take some time and you will be asked if you want to continue.

If you say Yes the validation will proceed and when it is complete, if any invalid items have been found, the system will display a box that lists a summary of them along with the options to [Save Details] or [View Results] as above.

Note: this facility is only available to System Managers - see User access levels.

Revalidating data to be exported

When data is exported e.g. for importing into another Recorder system - see Export for details - it is revalidated prior to export. If invalid items are found a message saying 'There are invalid records in the data you are about to export. These records will not be exported. Do you wish to continue?' If you say Yes the export will be done then, regardless of whether you said Yes or No, a filter will be loaded to enable you to see the invalid items.

 

Note: selecting an item and clicking on the * on the numeric keypad on the keyboard will expand the item in full.
