Import Wizard - matching data

The data in a number of the column types that you can select when importing observations need to exist in the appropriate tables in the database prior to adding the observations to the database:

  • Observer(s) and Determiner Name must be in the list of individuals in the Names and Addresses list - for more information see section below.

  • Species Name and Associated Species must be in the Taxon dictionary - for more information see section below.

  • Location must be in the location hierarchy or stored in Location Name - for more information see section below.

  • Abundance Data - the measurement qualifiers in this column type must be one of the qualifiers from a measurement type that is available on taxon occurrences - see the measurement type term list.

  • Association Type must be in the Relationship Type term list.

  • Biotope must be in the Biotope dictionary.

  • Determination Reference and Publication Reference must be in the document list.

  • Determination Type must be in the Determination Type term list.

  • Determiner Role must be in the Determiner Role term list.

  • Record Type must be in the Record Type term list.

  • Sampling Method must be in the Sample Type term list.

  • Specimen Type must be in the Specimen Type term list.

  • Substrate must be in the Substrate term list.

Note: the supported columns above are available in Recorder 6 up to and including version 6.16. However, the Import Wizard has been designed in such a way as to make it comparatively easy to modify it to import additional fields so subsequent versions are likely to support additional column types. Check what is currently available by selecting <more options> in the drop down box for selecting column types.

Following the page where you enter the required data that is not in the import file, Recorder will display a page for each of the above column types that is present. In most cases, it will automatically check whether the data are already present in the database, then display a table which lists all the distinct values from the file being imported, and the match it has found. On all the pages, you won't be able to proceed (the [Next] button will be inactive) until you have specified what is to be done with all the unmatched items. When a [Make New Entry] button is present one option is to select the unmatched item and click [Make New Entry]. Alternatively, [Make New Entries] can be used to create new entries for all unmatched items in one operation. This will add the selected item(s) to the relevant table(s) in the database when the button is clicked (the observations themselves aren't added until later in the process). Another option is to manually match the item concerned to an entry that already exists, e.g. Kate Wood could be matched to Katherine Wood; a measurement qualifier of 'males' could be matched to 'Male'.

Note: Indiscriminate use of [Make New Entry] and [Make New Entries] can lead to unnecessary duplication in your Recorder database, e.g. you could easily end up with both Kate Wood and Katherine Wood in your Names and Addresses list when they are the same person.

To match an item manually, or to change the match that Recorder has returned:

  • Click in the Recorder Matches box for the item you want to match or re-match.

  • If a drop-down arrow appears, use it to select an existing item.

  • If a Get button appears use it to select and Return the required item as normal, or use one of the other methods for Getting things.

  • In some of the matching pages you can type some matching text in the Recorder Matches box and click <Enter>. If there is only one match it will be returned. If there is more than one, a Find box will appear listing all the matches inviting you to select the required one. In the case of locations, if you choose Location Type, Spatial Reference and/or File Code using "Extra Location Search Columns" on the General tab in Tools - Options, then the selected types of data will by appear alongside the location names listed in the Matches box.

To delete a match, click in the Recorder Matches box containing the item you want to delete and press the <Delete> key.

When all the items are matched, if required, click [Next] to move onto the next page. On the last matching page this will cause Recorder to automatically move on to the Analysis and import phases.

Recorder remembers matches in a series of tables in the database called IW_Matched_... In subsequent imports, these will be used when the system is looking for matches for the imported data to save you having to do so many manual matches. They are highlighted in pale green to distinguish them from automatic matches and to encourage you to check that they are correct for the current import.

The [Commit Matches] buttons on the matching pages of the wizard enable you to control when matches are stored in the database for future use. If, for example, you are working on a large import that requires a lot of manual matching but you run out of time to complete it, you can click [Commit Matches] to save the matches you have done so far before cancelling the import. This will enable you resume the import more quickly as the matches already done will be remembered.

For help with these stages of an import, see the text on the left of the page. The [Next] button won't become active until all the issues are resolved for the current page.

Note: if you open other Recorder windows while using the wizard, you can return to it by selecting Window - Import Wizard from the main menu. If you add any data in the other window that you want to use in the wizard, you may need to use the [Previous] button then the [Next] button to effectively refresh the matching page.

The general principles of the matching pages are the same, although there are some important differences for Observer(s)/Determiner, Species and Location matching:

Observer(s) and Determiner matching page

If the import file contains Observer(s) and/or Determiner Name columns, this page is displayed, and the Imported Data column will list all the observers and determiners in the import file. The Recorder Matches column will contain the observers and determiners from the list of individuals in the Names and Addresses list that match those in the import file, and any remembered matches (shaded pale green). It is advisable to check the remembered matches.

Note: the import wizard can import multiple observers for an observation but only one determiner. Hence, if the Observer(s) column contains multiple names they will be split into multiple entries in the Imported Data column - see examples, but if the determiner column in the import file contains say "Thomas Jensen, Roger Little", a cell in the Imported Data column will contain the same text. One of them will need to be chosen as the determiner for the observation.

For any unmatched observers and determiners the options are:

  • Manually match the observer/determiner to an existing one, as described above.

  • Select the unmatched item and click [Make New Entry] to create the entry in the the list of individuals in the Names and Addresses list.

  • Click [Make New Entries] to create new entries for all unmatched observers and determiners in one operation.

When Recorder matches are completed for all the names listed, click [Next] to move onto the next page.

Species matching page

Matching Species and Associated Species are done on separate pages. The first time this page opens, the Imported Data column will list all the names from the column type Species Name but no matches will be listed until you trigger a search. After you have clicked on the Search botton Recorder will populate the Recorder Matches column with the matches it finds in the selected checklist. If the import file contains an Associated species column, the page will open again, after the Species Name matching is complete, listing all the names from that column.

When matching species, you need to specify which list in the taxon dictionary to use to find the matches:

  • Click the drop-down arrow beside Checklist to use for search and select the required list. Note that All preferred lists is available at the top of the drop-down list. If this is used the search is done across all the preferred lists, as in other parts of the system, e.g. in the taxon dictionary itself.

  • Click the [Search] button and wait for the search to complete.

  • Check remembered matches (shaded pale green).

  • Match any unmatched items manually, as described above. If there are any you can't match, you could return to the Column type selection page and remove the rows that contain them for processing later (species many need to be added to the taxon dictionary). If Auto-complete search in Import Wizard is ticked on the General tab in Tools - Options, then when you use the Get button to find matches for species in the import wizard, the Find Taxon dialogue will open with the Search Text box populated with the species name from the Imported Data column. You can then edit the Search Text to help find a match. If you also have Enable partial name search for taxa ticked you will have more options for editing the Search Text, e.g. you will be able to remove characters from the end of the genus. This will often speed up the matching process.

Repeat the process with other lists, if necessary.

When all species have been matched, click [Next] to move onto the next page. On the last matching page this will cause Recorder to automatically move on to the Analysis and import phases.

Note: for remembered species matches, the checklist is also remembered to help avoid problems with species like Redshank (plant and bird). However, this means that if, for example, a plant is matched when the LEPIDOPTERA list is selected at the top this match won't be remembered next time the wizard is used if the VASCULAR PLANTS list is selected. It will only be remembered if the LEPIDOPTERA list is selected.

Note: if you have matched all entries and the [Next] button is still greyed out, try clicking on another box. Changes to a box are not registered until you click away from it.

Note: for some lists searching the dictionary will be noticeably faster if the dictionary details are hidden.

Location matching page

If the import file contains a column given a type of Location (not Location Name), this page is displayed, the Imported Data column will list all the names from the Location column and the Grid Reference column will list entries from the Grid Reference column in the import file. Note that the contents of the Location Name column, if present, will not be included. See Location and Location Name for information on the distinction between the two. Recorder Matches will contain the locations from the Location hierarchy that match those in the import file, and any remembered matches (shaded pale green). It is advisable to check the remembered matches. The Centriod column will contain the central spatial references for the matched locations.

For any unmatched locations the options are:

  • Manually match the location to an existing one, as described above.

  • Select the unmatched item and click [Make New Entry] to create the entry in the location hierarchy. If there is more than one grid reference for the location in the import file you will need to decide which to use as the site centroid or central spatial reference for the new location. To do this:

    • Click in the corresponding Centroid box.

    • Type in a grid reference for the site centroid or click on the drop-down arrow that appears and select one of the grid references from the import file.

    • Click [Make New Entry].

  • Click [Make New Entries] to create new entries for all unmatched locations in one operation.

  • Tick Place any remaining unmatched locations into location name. When this is selected the location information from the import file will be stored in (Survey) Event Location Name and Sample Location Name for any unmatched locations; the observations will not be linked to locations in the location hierarchy. If location name is also present the two will be concatenated.

Note: you have the option to Use the first listed grid reference when making new entry by ticking the box at the top.

If you use [Make New Entry] the new location will be added at the top level in the location hierarchy although it can be moved later, if required.

Note: if you want to add the location directly into the location hierarchy so that it goes in at the appropriate level rather than at the top level, simply click on the Get button that appears when you click in the Recorder Matches box to open the location window and add it, then select the Return button to return it to the location matching page.

When you have taken the appropriate actions for all the unmatched locations, click [Next] to move onto the next page. On the last matching page this will cause Recorder to automatically move on to the Analysis and import phases.
