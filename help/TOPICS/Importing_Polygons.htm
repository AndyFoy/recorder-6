Importing polygons

Sets of polygons created in other systems such as GIS like MapInfo and ArcView, can be imported into Recorder. For a list of the formats that Recorder will acccept see Map formats. Polygons are imported onto polygon layers each of which have their own pattern and selected and unselected colours. To add a new polygon layer to import a set of polygons onto see Adding a map layer.

Note: The Watsonian vice-county boundaries for Great Britain are available for download from the NBN for use as polygons in Recorder - see Useful things - Mapping tools on the NBN web-site.

Note: A variety of boundaries are available for downloading from the Natural England web-site, e.g. NNR (Natural Nature Reserve) boundaries for a specified 100 KM square.

To import polygons:

  • Select the polygon layer onto which the polygons are to be imported using the down arrow in the Current Polygon Layer box (bottom left corner of the map window).

  • Select Map – Add Boundary – Import from the Main menu, or use the drop down arrow beside the boundary icon on the Active Window Toolbar to select Import onto layer.

  • A standard Windows file dialog will open. Locate and open the vector file which contains the lines or polygons you wish to import and click [Open]. When the file has been read you will see the Boundary Import dialog:

Match boundaries to existing locations

Select No to import the polygons without matching them to locations in the location hierarchy. Click [Import] to proceed with the import. Click [Cancel] to cancel the polygon import.

Select Yes to import the polygons and match them to locations in the location hierarchy.

Boundary Attribute Field

Note: This will be greyed out unless Yes is selected for the options above.

Select <Manual Match> to import the polygons and match them manually to locations in the location hierarchy. Click [OK] to proceed to the next dialog.

To match the polygons to locations using one of the boundary attributes in the import file, use the drop down arrow to make your selection.

Location Field

Note: This will be greyed out unless Yes and an attribute are selected for the options above.

Use the drop down arrow to select the Location Field from locations in the location hierarchy you want used to match to the selected Boundary Attribute Field. The options are:

    • Location Key

    • Location Name
    • Field Code

  • If Yes is selected for Match boundaries to existing locations, NNR_NAME for Boundary Attribute Field and Location Name for Location Field,you will see the Location Match dialog:

Boundary Count

A count of the number of polygons in the import file

Matched Locations

A count of the number of polygons that have been matched to locations in the location hierarchy.

Boundary Attribute Field

If a specific Boundary Attribute Field was selected in the previous dialog its name will appear in this box.

If <Manual Match> was selected, use the drop down arrow to select the Boundary Attribute Field you want to use to help match the polygons to locations.

The values for the selected attribute from the import file will appear in the middle column in the table below alongside the polygon identifiers. You can switch to different attributes, e.g. grid reference, to help with the matching.

Location Name

If a specific Boundary Attribute Field was selected in the previous dialog, Recorder will search the location hierarchy for locations with the matching Location Fields (Location Key, Location Name or Field Code) and display the matching entries. You will be able to manually match any unmatched polygons, as specified below, if required.

If <Manual Match> was selected in the previous dialog, you will be able to manually match polygons, as specified below.

    • To match a polygon to a location, click in the Location Name box for it. The Get button appears and can be used it to select and Return the required location. Alternatively, type some text and press <Enter> as in Finding things.

If a location doesn't exist you can use Make new Entry to create it in the location hierarchy.

After matching all the polygons you can, you have the option to Make New Entries for all the remaining unmatched polygons.

Note: Clicking on a column heading will sort the rows into ascending sequence; clicking again will sort them into descending sequence.

Note: Unmatched polygons will be imported but not linked to locations.

  • Click [Import] to proceed with the import.

  • Click [Cancel] to cancel the polygon import. No polygons will be imported.

Note the progress bar and allow time for the polygons to be imported. The import process will convert the files you are importing into the format required by Recorder and store the resulting information in the files that constitute the polygon layer in the Object Sheet folder in the Recorder folder.

Provided the import is successful, the lines and polygons contained in the file will be drawn on the map. They will have the properties (pattern, selected and unselected colours) of the layer to which they have been added. Users can change these properties, but any change affects all polygons on that layer.

When polygons have been linked to locations, selecting the polygon on the map will display the location name in the Location box at the bottom of the map window. Also, Find On Map can be used from the location hierarchy to select a polygon on a map.

TIP: You may not necessarily see all (or any!) of the imported objects, because they could be drawn on a part of the map that is not currently visible because you have zoomed in on another area. Pan or zoom out until the relevant area is visible. If you are not sure where the objects will be located, zoom to extents will return you to a display of the entire map.
