Reporting Bugs

If you encounter a bug or error in the Recorder software or documentation, the developers would like to know about it. A "bug" is defined as "any problem found in the software that is solved by changing the source code of the software".

Bugs and unhandled errors should be reported on the Recorder Forum, part of the NBN Forum. Upgrades to fix bugs will be made available via the Recorder web site, www.jncc.gov.uk/recorder.

When reporting a bug, please try and provide as much detail as possible on the circumstances, the sequence of actions that led to the problem, the symptoms that occur and the operating environment. The following checklist may be useful:

  • The version of the Recorder software and any components it is using. Use Help - About Recorder 6 from the main menu to obtain this information.

  • The operating system you are using, e.g. Windows XP, Vista, Windows 7, Windows Server 2003.

  • The version of SQL Server you are using - see Installing on a single machine for the list of versions supported.

  • The specification of the machine - what sort of CPU, how much RAM, whether it is 32-bit or 64-bit, etc.

  • The sequence of actions you performed which led to the error. Please try and describe these as accurately, and in as much detail as possible.

  • A description of the problem including a note of any error messages that were reported. Note: if you get an unhandled error, from Recorder 6, it saves the details in a text file. Include a copy of this file if you are reporting a bug.

  • Is it reproducible? (If you carry out this sequence of actions again, do you get the same error?). Errors that are not reproducible can be very hard to track down and fix!

  • Depending on the nature of the problem, it may be helpful to supply copies of the data you were using at the time, e.g. if you encountered an error whilst doing an import, then it would be very useful to supply a copy of the file you were trying to import when the problem arose.
