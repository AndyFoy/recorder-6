Spatial References

The geographical co-ordinate for a position on the Earth’s surface can be specified in a wide variety of different systems. Some (like Latitude and Longitude, and UTM) are global, whilst others are local in scope (like National Grid systems).

Recorder supports multiple spatial reference systems and four are supplied:

  • Ordnance Survey National Grid for Great Britain.

  • OS National Grid for Ireland.

  • Latitude and Longitude.

  • Universal Transverse Mercator (UTM).

Additional systems can be supported by Add-Ons.

Spatial references can be entered using any supported system and, internally, all spatial references are stored as latitude and longitude co-ordinates. See How spatial references are stored for more details.

The currently selected spatial reference system is used to format spatial references for display on screen and for reporting. However, it is not always possible to convert the lat./long. co-ordinate into the currently selected format. For example, suppose Irish grid references and the lat./long. co-ordinates for offshore observations had been entered, and the current system was 'OS National Grid for Great Britain'. These references may be outside the area covered by the GB national grid and, therefore, cannot be shown as GB grid references. In these cases, the original spatial reference is shown as BEYOND SYSTEM LIMITS.

Another piece of information which is stored alongside the spatial co-ordinates, is the way in which the information was obtained (spatial reference qualifier).

Examples include:

  • Estimated from map.

  • Obtained from internal map.

  • GPS.

  • Site centroid.
