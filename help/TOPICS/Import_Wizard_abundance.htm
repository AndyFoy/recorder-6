Import Wizard - abundance data

To import abundance data via the import wizard you can select the following column types:

  • Abundance Data and/or

  • one of the Taxon Occurrence Data options

Abundance Data

An Abundance Data column can contain a number of sets of abundance data separated by commas or semicolons (;), e.g. 1m,2f,3j. The first part of each field will appear in the 'Data' column of the Measurements for the taxon occurrence. The second part will appear in the 'Qualifier' column of the Measurements. The wizard will ask you to match the qualifiers to Measurement Qualifiers from the Measurement Type term list items that are available on taxon occurrences. This option can be used in combination with Taxon Occurrence Data.

Note: An Abundance Data column can contain characters like -, *, + and #, but not $.

Note: If a set of abundance data ends in a character that isn't alphabetic or a ), a bracket, it is assumed that it consists of Measurement Data only so the whole set goes into Measurement Data and Measurement Qualifier is set to "None". This enables values like "10 - 20" and "> 150" to go into Measurement Data.

Note: If the data to go into Measurement Data is too long for the field in the database, i.e. greater than 20 characters, it will be flagged as an error and will have to be shortened before you can proceed.

Taxon Occurrence Data

This option is available under <more options>. Clicking the beside it opens a list of every possible combination of Measurement Qualifier and Measurement Unit from the Measurement Type term list that is available on taxon occurrences. This is the same list that is available on the Measurements tab when constructing a recording card.

Any of these can be selected as the column type for a given column. Then, data like 2 adults and 3 juveniles could be input with

2 in the Abundance of Adult (Count) column and
3 in the Abundance of Juvenile (Count) column

These options can also be combined with an Abundance Data column so that Taxon Occurrence Data columns can be selected for commonly entered values but an Abundance Data column can be selected for anything else that is entered.

Note: Taxon Occurrence Data columns can contain characters like -, *, + and #, but not $.

Note: If allowed values have been added to the measurement unit in the term lists for the unit selected the contents of this column will be validated against them.
