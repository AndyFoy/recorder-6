Biotope Occurrence General Tab

This page can be used to record basic information about the observation, its quality and status.

Surveyor's ref

Free text. Surveyors frequently use some form of reference number to identify particular observations. Such references can be stored here to help locate material, such as the original field notes, in a conventional filing system.

Comments

RTF memo. Any comment.

Determination Type

The determination type from the preferred Determination is displayed in this box. It can be changed by editing the observation and using the drop down box to select a different value. The available values are those from the determination type term list. Users can add terms to this list if necessary.

See Determination verification for details of how this is used to set the Verified flag on the biotope occurrence.

Note: determiner types can be hidden so that they can't be used for data entry - see Determiner type for details.

Checked

Has this observation been checked? – tick for Yes; defaults to unticked for No.

'Checked' means that the information stored in the database has been compared with the original (e.g. checked against the record card), preferably by somebody other than the person who made the data entry, and is an accurate transcription. It is not a guarantee, however, that the original information was correct.

Note: If an observation is NOT flagged as checked:

  • It cannot be exported.

  • It will not appear in reports unless you specifically indicate that unchecked records should be included (Report Wizard – Additional filters – Constraints.

Digitised

Has the habitat patch been digitised? – tick for Yes; defaults to unticked for No.

Verification

This indicates whether the observation is:

  • Not verified

  • Failed/pending verification

  • Passed verification

It is set according to the determination type on the preferred Biotope determination as specified in Determination verification.

Note: If an observation is flagged as Failed/pending verification:

  • It cannot be exported.

  • It will not appear in reports unless you specifically indicate that records flagged as Failed/pending verification should be included (Report Wizard – Additional filters – Constraints.

Note: you cannot change an item unless you have sufficient access rights and your copy of Recorder is the custodian of it - see Editing an item for more information.

Tip: To display unchecked, zero-abundance or failed/pending verification observations see Display Unchecked records.
