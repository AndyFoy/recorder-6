Taxon Occurrence General Tab

This tab records basic details about the occurrence, including the type of record, whether it is confidential and any comments. The information from the measurements page is summarised, together with whether or not it has been verified from the determination page.

 

Surveyor's Ref

Free text. Surveyors frequently use some form of reference number to identify particular observations. Such references can be stored here to help locate material in a conventional filing system, or to associate the record with other stored material.

Record Type

Type of record selected from the record type term list. Users can add terms to this list if necessary. The sampling method is given in the sample record to which this occurrence is attached, but this field can be used to store additional detail. For example, the sampling method might be a transect walk, but the observations made during that walk could include a mixture of field observations and records based on signs or specimens that were retained for later examination. 'Record type' could be used to distinguish these observations.

Substrate

Substrate selected from the substrate term list. Users can add terms to this list if necessary.

Provenance

Free text, but with a list of frequently used values from which an entry may be picked. Used, for example, in BSBI plant recording schemes to note whether a specimen was planted or native.

Comments

RTF Memo. Any comment. This could be used to note observations of behaviour or anything unusual about the occurrence.

Count

Read-only summary of information from the Measurement page.

Determination Type

The determination type from the preferred Determination is displayed in this box. If this is changed a new determination will be added to the taxon occurrence. Clicking on the drop down box and selecting a different option will open the Dets. tab in edit mode with:

  • Taxon set to the taxon from the preferred determination

  • Determiner set to the name of the person currently logged on, as displayed on the Welcome window
  • Role set to 'Specialist'
  • Type as selected
  • Date set to today's date

To save the new determination, make any changes required, then click the button to save your changes. Click the button to abandon your changes.

The available values for determination type are those from the determination type term list. Users can add terms to this list if necessary.

See Determination verification for details of how this is used to set the Verified flag on the taxon occurrence.

Note: determiner types can be hidden so that they can't be used for data entry - see Determiner type for details.

Confidential

Should this observation be treated as Confidential? – tick for Yes; defaults to unticked for No.

Note: If an observation is flagged as Confidential:

  • It cannot be viewed by users with an access level below 'Full Edit (Own data only)'.

  • It will not be exported unless Enable export of confidential occurrences is ticked on the General tab in Tools - Options. See also Export management.
  • It will not appear in reports unless you specifically indicate that confidential records should be included (Reports – Wizard – Additional filters – Constraints).

Checked

Has this observation been checked? – tick for Yes; defaults to unticked for No.

'Checked' means that the information stored in the database has been compared with the original (e.g. checked against the record card), preferably by somebody other than the person who made the data entry, and is an accurate transcription. It is not a guarantee, however, that the original information was correct.

Note: If an observation is NOT flagged as Checked:

  • It will not be exported in certain circumstances - see the notes in red in Exports.

  • It will not appear in reports unless you specifically indicate that unchecked observations should be included (Reports – Wizard – Additional filters – Constraints.

Verification

This indicates whether the observation is:

  • Not verified

  • Failed/pending verification
  • Passed verification

It is set according to the determination type on the preferred Taxon determination as specified in Determination verification.

Note: If an observation is flagged as Failed/pending verification:

  • It will not be exported in certain circumstances - see the notes in red in Exports.

  • It will not appear in reports unless you specifically indicate that observations flagged as Failed/pending verification should be included (Reports – Wizard – Additional filters – Constraints).

Note: you cannot change an item unless you have sufficient access rights and your copy of Recorder is the custodian of it - see Editing an item for more information, but you can add a new determination, if you have sufficient access rights, even though you are not the custodian.

Tip: To display unchecked, zero-abundance or failed/pending verification observations see Display Unchecked records.
