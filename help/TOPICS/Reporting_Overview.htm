Reporting overview

The reporting facilities available with Recorder are designed to enable you to report on your data in a variety of ways, as outlined below.

The Report wizard

The Report Wizard is designed to provide an easy-to-use way of answering three basic questions:

  • What observations have been made for given taxa?

  • What observations have been made for given biotopes?
  • What taxa and/or biotopes have been recorded at a particular place?

The Report Wizard leads you through the steps necessary to answer these questions. The results of the Wizard include a query (written in SQL - Structured Query Language) and, optionally, a report template (the layout used to print information) or the instructions for creating a Snapshot. Two standard templates are available to provide a pre-defined layout in which the answers to these basic questions can be presented, or a built-in Report Designer can be used to design report layouts. The query and template can be saved for future use, as can a snapshot. Therefore, once a particular report has been saved, it can be re-run at any time using Reports - Run, and it will report on the current, up-to-date state of the database. Saved reports can also be given to other Recorder users so that the same reports can be produced from a number of different copies of Recorder.

For more information see Report Wizard.

Standard Reports

A number of reports are available as standard facilities in the system:

  • Occurrences for Places Report - lists the species and habitat observations for a selected location.

  • Places for Occurrences Report - lists the observations for a selected taxon in the taxon dictionary or the observations in a survey, survey event or sample in the observation hierarchy.

To run the standard reports - see Running Reports.

XML reports

Recorder comes with an extensive selection of XML reports, a powerful addition to the reporting tools in the system. If you require a report that can't be produced using the other reporting facilities you may find that it is already available as an XML report. Alternatively, you will be able to develop it yourself, provided you have access to the necessary computing skills. These reports can be run from within your Recorder system using one of the methods described in Running Reports. They can also be designed to run from a selected item in a Recorder window.

For more information on these reports, see XML reports supplied with Recorder. For introductory information on developing XML reports please click here.

Other Reporting Facilities

As well as the reporting facilities described above you can also report directly on the Recorder database. This is a SQL Server database called NBNData_Data.MDF which, in a standard standalone installation, will be found in:

C:\Program Files\Microsoft SQL Server\MSSQL.1\MSSQL\Data. Also in this folder will be the associated log file, NBNData_log.LDF.

As specified in Accessing the database from other software, JNCC have also made an Access database - nbndata.mdb - available to enable you to query the Recorder database using Microsoft Access. If you were accustomed to using Access to query your Recorder 2002 database, you will be able to transfer your queries to this version of nbndata.mdb which links to the Recorder SQL Server database.

If you have the necessary SQL Server tools, or any other software capable of accessing a SQL Server or Access database (e.g. a Geographical Information System or a dedicated report writer) you will be able to use them for reporting and analysis of the Recorder database.

Note: The Recorder database structure is very complex and will require time and effort to master. It is based on the NBN data model which is documented in the Documentation Wiki on the Recorder web site, www.jncc.gov.uk/recorder.

SQL Reports

In Recorder 2002, there was an add-in called SQL Reports which provided a basic library of pre-set reports written in SQL. This library could be added to by writing your own SQL reports in Microsoft Access. In Recorder 6, this functionality has been superseded by XML reports.
