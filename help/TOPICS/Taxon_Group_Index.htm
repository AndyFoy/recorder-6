Taxon Indexes

The Taxon Indexes let Recorder make use of the complex taxon dictionary more quickly and effectively.

Index Taxon Group

This supplies information about which taxa are included within higher taxa in the taxonomic hierarchies in checklists. For example, it tells the system which families, genera, species and sub-species are in the order "Diptera". This allows Recorder 2002 to report on higher level taxa easily and efficiently.

You can:

  • Use higher level taxa in the Report Wizard and, optionally, have all the taxa within them included in the report. See Report Wizard - Taxa Selection for details.

  • Drag a taxon onto the Map window to get a distribution map for all the taxa within it. Again, you will be given the option of mapping all taxa within the group, or only including observations for which the taxa was specifically given as the preferred determination.

  • Set up your own higher-level groups which contain an arbitrary collection of taxa at any level. For example, you might want to set up a group "Waterbeetles" which includes a mixture of families, sub-families and genera. This is achieved through the edit taxon group page. You can then drag "Waterbeetles" on to the Map window to produce a distribution map of all waterbeetles, or include it in the Taxon Selection of a report.

Index Taxon Name

This supplies a number of very important elements for each taxon including it's sort order, recommended name, common name etc. Index Taxon Name is inherent to the way in which Recorder translates between items on different checklists and provides standard names and sort orders for entries.

Making use of the indexes

If you import a new checklist (e.g. from an XML file) or through an update you will need to rebuild the indexes

Select Tools - Database Tools - Rebuild Taxon Group Index from the Main menu.

Select Tools - Database Tools - Rebuild Taxon Name Index from the Main menu.
