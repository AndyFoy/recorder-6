Polygon selection

You see this screen if you selected the Restrict to one or more Polygons option in the starting screen of the Wizard. It is used to select the polygons on the map that you want to use to select observations (the report will then include observations which fall within the selected polygons).

Filter report by selected polygons

This list initially shows the names of your polygon layers. Double-click on a name, or click the button, to show the polygons on that layer. If you have linked a polygon to a location, you will see the name of the location, otherwise it will be shown as "Unlinked Polygon [ID: nnn]". You can select a polygon by clicking the check-box (so that it is ticked). Click again to unselect (which removes the tick).

Find Polygon on Map

Press this button to open the Map window. You can then select a polygon on the map by clicking on it. Press return data [F9], to return to the Report Wizard. The polygon will now be ticked in the list.

Predefined Filter

The list contains sets of polygons you have previously saved using the Save Filter As facility. Choose an item from this list and the list of polygons that was saved will be ticked.

Save Filter As

You can save the selection you have made for use on a future occasion. You will be prompted for the name of a file. It is saved in the directory nominated in the Polygon Filters: entry in Options - File Locations.

Include overlapping grid squares in report results

Include grid squares that overlap the polygon boundary in the report results.

Exclude overlapping grid squares in report results

Exclude grid squares that overlap the polygon boundary in the report results.

Until the Include/Exclude overlapping grid squares in report results options were introduced in 2008, an observation was selected for the report if its spatial reference fell within one of the selected polygons. Grid references like 10km squares, 1km squares or tetrads (2km squares) are represented internally by the spatial reference of the bottom left-hand corner. So, an observation for a 1km square, for example, was considered for the report if its bottom left-hand corner was within the polygon and irrespective of whether any other part of the 1km square was within the polygon. Now you have the option to specify whether you want grid squares that overlap the polygon boundary included or excluded from the report results.
