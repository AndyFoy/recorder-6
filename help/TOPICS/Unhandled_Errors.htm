Unhandled Errors

If users make mistakes when using Recorder 6, the system will often display an appropriate message telling them what is wrong. They can then correct the problem and continue using the system. However, from time to time, problems occur that the system cannot handle. In this case an exception message like this appears:

As stated in the information at the top of the message it will be saved as a text file in the specified location. Multiple files can be saved and the system automatically deletes any over 7 days old. Much of the information in the message will be unintelligible to most users but some of it can help developers diagnose and fix the problem. The exception message at the end of the screen shot can sometimes be used to find more information about the problem that has occurred.

If you get a message like this, for example, when you are entering data, please check whether the system has done what you asked it to do. Sometimes it will have done what you wanted it to do and there is no need to re-enter the data.

If the application will not continue, try closing it, re-opening it and doing what you were trying to do more carefully.

If the unhandled error continues to occur:

  • check whether the exception message is listed in the Resources - Known Issues and Errors section of the Recorder web site, www.jncc.gov.uk/recorder.

  • try searching the Recorder Forum, part of the NBN Forum, for other occurrences of the same problem. Googling ‘NBN Forum xxx’ where xxx is an extract of the exception message will sometimes find other reports of it.

  • report it on the Recorder Forum, part of the NBN Forum. When reporting problems please provide as much information as possible - see Reporting bugs.

Access violation messages, as in the screen shot above, are generic messages which aren't very helpful for users but often the exception message will contain something that can be used to find more information which may offer a solution or work around.
