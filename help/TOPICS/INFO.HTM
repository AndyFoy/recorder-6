Meta-data

Recorder logs a selection of meta-data for most items in the database, e.g. who created them and when, who last changed them and when - see Information to track changes to items. This meta-data can be viewed using the Show Metadata icon along with the key, or unique identifier, for the selected item and, if it has been imported, the Site (or Copy) Id of the copy of Recorder it was imported from. Also, if ownership/custody has been transferred from another copy of Recorder, the Site Id of the copy of Recorder that currently owns it will be displayed.

The unique identifiers for items stored in the Recorder database, sometimes refered to as the NBN Globally Unique Identifiers, are 16 characters long. The first 8 characters are the Site Id of the copy of Recorder to which the item was originally input. The second 8 characters are a running number made up of upper case letters (A..Z) and digits (0..9), i.e. 36 possible characters. The LAST_KEY table in the database records the last running number used to form a key for each table. When an item is added to the database, the running number for the relevant table is read from LAST_KEY, incremented by one, and combined with the copy ID to form a unique key for the new item. At the end of the process, the entry in LAST_KEY is updated.

For the taxon and biotope dictionaries, the Show Metadata icon displays meta-data for the selected item and for the list or classification it is on.

To view meta-data:

  • Select the item for which meta-data is required.

  • Select Edit - Show Metadata from the main menu, or

  • Click the Show metadata button in the active toolbar.
