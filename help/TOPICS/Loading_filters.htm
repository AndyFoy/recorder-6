Loading filters

Filters that have been saved when revalidating or importing data can be loaded at any time so that the data concerned can be viewed.

To load a filter:

  • Select Tools - Load External Filter from the main menu.

  • Navigate to where the filter has been stored and select the required one. By default, filters saved by Recorder will be stored in the User Files folder in your Recorder folder or in \User Files\Reports\Output. For a standard standalone installation with Windows XP, this will be found in C:\Program Files\Recorder 6\User Files.

  • Click [Open] to load the filter.

When an external filter file is loaded, the filter is applied to the relevant windows, i.e. observations, locations or one of the other main data entry windows. Expand the items as normal by clicking on the beside them and view different windows in the normal way. Any item actually part of the filter is shown highlighted in a different colour, so that parent items which are displayed but not part of the filter can be discerned. Floating the cursor over a highlighted item displays some details, if present in the filter file.

Note: selecting an item and clicking on the * on the numeric keypad on the keyboard will expand that item in full.

To remove a filter:

When a filter is applied to a window (either by loading a filter, using the Simple filter tool, or as a result of using the [Related data] button from another window), a [Show All] button appears towards the bottom right-hand corner of the window.

  • Click [Show All] and the filter will be removed.

  • All data in the window will be accessible again via the hierarchy, or list, in the left-hand panel.

  • The item that was selected when the [Show All] button was clicked will be highlighted.

  •  The [Show All] button will disappear again.

Filters can be saved when:

  • revalidating data - [Save Details] on the results window allows you to save the details of the invalid items in a file that can be loaded as a filter to enable you to view them at any time - see Validating data.

  • importing data

    • [Save Details] on the Invalid items tab at the analysis stage allows you to save the details of the invalid items in a file - see Invalid items. If the import came from another copy of Recorder this file can be used to help resolve the problems in that system.

    • Save the list of imported records as a filter file so that you can review them later on the completion message at the end of the import allows you to save the details of the imported items in a file that can be loaded as a filter at a later date - see Analysis and import.

Filter file format

The external filter files created by Recorder are text files with an extension of .ref. The format is:

  • First line: Description: followed by details of the process that created the filter.

  • Subsequent lines: for each item in the filter, a line with the following tab separated values:

    • Table name from the database

    • Primary key
    • Details - optional - text that in the case of revalidations will explain why the item has been identified as invalid. This is the message that will appear when you load the filter and float the cursor over the highlighted items.

 

Note: filters of different types can be applied to the main windows using the filter icon on the active window toolbar - see Simple filter.
