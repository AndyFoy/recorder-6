Dictionary Upgrades

Upgrades for the dictionaries will periodically become available and will need to be included in your Recorder database so that you can enter observations using up-to-date taxonomic checklists and biotope classifications. Taxon dictionary upgrades may be required, for example, as a result of new species being recorded for Britain, or in response to changes resulting from research into taxonomy.

When a dictionary upgrade for Recorder becomes available it will be announced on the Recorder Forum, part of the NBN Forum, which can also be accessed from the Recorder web site, www.jncc.gov.uk/recorder. You are advised to check this from time to time for news of upgrades and other information about Recorder.

To upgrade the dictionaries you need to use the Dictionary Upgrade Kit which first became available in version 6.9 of Recorder. It was introduced so that changes to the dictionaries could be done independent of changes to the software. Prior to version 6.9, upgrades to the software and all the dictionaries were combined.

Dictionary upgrades come in the form of upgrade files or scripts that need to be placed in the Dictionary Upgrade Kit's Scripts folder prior to running the DictionaryUpgrade.exe file. The dictionary upgrade does the database changes specified in the files in the \Dictionary Upgrade Kit\Scripts folder. When an upgrade completes, it stores the most recently applied file name (excluding the extension) in the Recorder database. This is the dictionary version that can be displayed using Help - About Recorder 6. It is used to prevent you from skipping upgrades or running them out of sequence.

Users are advised to backup their database prior to upgrading via Tools - Database Tools - Backup Database or the SQL Server tools they use for backups.

Note: Whenever an upgrade is to be installed, it should be done through an administrator user account to avoid any potential problems due to limited user account access rights. The "sa" login or a trusted connection with database owner privileges over the Recorder database is required during the upgrade.

If you are upgrading a standalone copy of Recorder on your personal computer running Windows XP, the way you normally login will probably provide sufficient rights to run an upgrade, but if you are operating on a network this may not be the case and you may need to ask a system administrator to run the upgrade for you. If you are running Vista or Windows 7, you will need to request elevation by taking the actions specified below.

Note: If an upgrade adds large lists to the dictionary or makes extensive changes to existing lists, it may take a long time to run, however this doesn't apply for upgrades 0000000F to 0000000Y. In this case, you may be advised to run the upgrade overnight, although the runtime will depend on the power of your computer(s) and the speed of your network, where applicable.

A large dictionary upgrade will require a lot of processing power to run and therefore you may receive a message in the title bar of the "Recorder Dictionary Upgrade" window saying that the application is not responding, or it may appear blank. PLEASE DO NOT CANCEL THE UPGRADE AT THIS STAGE. Press <Ctrl-Alt-Del> and open Task Manager. Under the Applications tab the status of the "Recorder Dictionary Upgrade" will also be displayed as not responding, however, if you click on the Performance tab, the CPU Usage will probably be nearing 100% and the CPU Usage history line will be at the top of the grid box. Unless the CPU Usage is low and static, the upgrade is still running. If you run Recorder on a network, then you will need to monitor CPU usage both on the server and local machine to check that the upgrade is still running. The upgrade splits it's time between the two and therefore the local machine may become inactive whilst the server continues to process the upgrade.

Note: If you are using a free run-time version of SQL Server you need to keep an eye on the size of your Recorder database as large dictionary upgrades can add significant amounts of data to it. With MSDE the maximum database size is 2 GB, with SQL Server 2005 Express it is 4 GB, with SQL Server 2008 Express it is 10 GB. If data entry or a dictionary upgrade would take you over this, SQL Server simply doesn't do it.

To upgrade the dictionaries:

  • Download the dictionary upgrade kit from the Dictionary Downloads section of the Recorder web site, www.jncc.gov.uk/recorder, if this hasn't been done for a previous upgrade, or if a new version is now available. It is recommended that you store this in a folder outside your Recorder 6 folder(s).

  • Unzip the dictionary upgrade kit. Clicking on the ".zip" file will usually open the extraction software on your computer. You need to select the options that will unzip the selected file to a new folder in the same location and retain the folder structure when the file is unzipped (often the defaults). If extraction software is not available, you will need a suitable software tool, such as WinZip, to extract the files from the ".zip" file. The unzipped file should look like:

  • Download the required upgrade file(s) from the Dictionary Downloads section of the Recorder web site, www.jncc.gov.uk/recorder. It is recommended that you store them in a folder outside your Recorder 6 folder(s). Do not save them in the Scripts folder in the dictionary upgrade kit.

  • Unzip each upgrade file in turn and place the unzipped file(s) in the Scripts folder in the dictionary upgrade kit. The files you download are called, for example, Rec6Dict_0000000R_UnzipMe.zip, which contains the files 0000000R.ini and 0000000R.zip. It is 0000000R.ini and 0000000R.zip that need to be in the Scripts folder not Rec6Dict_0000000R_UnzipMe.zip. Depending on the nature of the upgrade, the scripts will be .zip files (usually accompanied by .ini files) or .sql files. If the Scripts folder contains any Rec6Dict_000000... files when dictionary upgrade is run, it won't run correctly and it can prevent subsequent upgrades from running.

Note: The downloaded files must be unzipped just once, so for example, 0000000R.zip must not be unzipped again as the upgrade kit is designed to cope with it in this format. Do not rename any of these files.

Note: For the upgrade to run correctly there must be a Scripts folder containing the scripts to be run in the Dictionary Upgrade Kit folder.

Note: You can run multiple upgrades at the same time but the Scripts folder must contain all the scripts from the one after the last one applied (see Help - About Recorder 6) to the latest one you wish to apply. The upgrade kit expects to apply the scripts in file name sequence. It will not run a script if the previous one hasn't been run and an appropriate error message will be displayed if an upgrade file has not been run, e.g. if the dictionary version is 0000000Q but the Scripts folder contains 0000000S.ini, 0000000S.zip and 0000000T.sql, then an error message will be displayed when the upgrade is run. 0000000R needs to be run before 0000000S and 0000000T.

  • Make sure that Recorder is not running.

  • Run the upgrade by:

    • Locating the DictionaryUpgrade.exe file amongst the files you unzipped using "My Computer" or "Windows Explorer", and double-clicking it, if you are running Windows XP, or by right clicking it and selecting Run as administrator, if you are running Vista, or by double-clicking it and answering Yes to the User Account Control question 'Do you want to allow the following program (DictionaryUpgrade.exe) from an unknown publisher to make changes to this computer? if you are running Windows 7.

    • Alternatively, select Start - Run from the Windows menu, type "\DictionaryUpgrade.exe" preceded by the full file path for the folder that contains it and click [OK]. Note that for Vista and Windows 7 this option works a little differently.

Note: If you are upgrading a network installation the upgrade needs to be run from a computer that operates as a Recorder workstation as the upgrade uses the registry settings on the workstation to locate the things to be upgraded.

  • You will be prompted to select the SQL Server system administrator login you would like to use to log on to the SQL Server. The options you chose will depend on how SQL Server was installed. They are:

    • "Use the following SQL Server system administrator login", for which you need to input a Username and Password. If you are running the system under MSDE accept the default, i.e. Username "sa" and Password left blank. For other versions of SQL Server, you will need to know the password that was specified when SQL Server was installed.

    • "Use my Windows Account to login (Windows Authentication)", the trusted connection option. Use this option if Windows Authentication or Mixed Mode Authentication was specified when SQL Server was installed. If you are running a standalone installation with Windows XP, Vista or Windows 7, it will be easiest to use this option.

  • When the upgrade is complete the message "Recorder upgrade complete" will appear.

Note: If, for any reason, a dictionary upgrade does not appear to have completed correctly, please seek help via the NBN Forum, or contact the supplier of your copy of Recorder, or any of the Recorder suppliers who are listed on the Recorder web site, www.jncc.gov.uk/recorder.

You are now in a position to use Recorder again although, if the upgrade contains changes to the taxon dictionary, you must rebuild the taxon indexes before you can use the new or changed entries in the taxon dictionary:

  • Select Tools - Database Tools - Rebuild Taxon Group Index from the Main menu, then

  • Select Tools - Database Tools - Rebuild Taxon Name Index from the Main menu, then

  • Select Tools - Database Tools - Rebuild Taxon Synonym Index from the Main menu, then

  • Select Tools - Database Tools - Rebuild Taxon Designation Index from the Main menu.

Note: Rebuilding the Taxon Designation Index must be done last as it uses the other indexes.

Note: Depending on the power of your computer(s) and the speed of your network, where applicable, each rebuild may take 5-10 minutes to run. On slower systems they may take somewhat longer.

Use Help - About Recorder 6 to check that the system has been upgraded.
