Entering data using a Recording Card

The recording card is an efficient method for entering multiple species observations for a location and date. The information on each card corresponds to a single survey event with one or more samples.

To enter data using a recording card:

  • Click Data Entry – Enter Species for a place.

  • In the sub-menu, select the recording card that you want to use. If there are no cards listed, you will need to create a new recording card (see creating a new Recording Card).

  • The Recording Card dialog box appears with the name of the recording card in the title bar.

  • The card is designed to allow keyboard data entry for speed.

The methods for finding and entering existing data on a recording card are the standard Recorder methods. For information on how to find existing data see Find overview.

Survey

Required.

Select an existing survey from the list.

Location

 

Optional. This is a reference to an item in the Locations window. To enter a location, use one of the methods described in Finding things.

If a location is entered, the spatial reference value is completed with the location's central spatial reference by default. However, this default value can be edited if an exact spatial reference is available.

Entering an exact spatial reference in the Spatial Ref field will not effect the location.

An entry in at least one of the Location, Location name or Spatial Reference fields is REQUIRED, i.e. some form of locality must be given.

See Location and Location Name for more information on the distinction between the two.

Location Name

Optional. A name for the sampling location which is stored as text in the Survey event and Sample records rather than a reference to an item in the Locations window. If you enter a Location, this field can be used to provide additional details - for example, describing exactly where the observations were made within a large site. It is intended to enable a more detailed, textual description of the location of the observations to be entered.

Date

Required. Enter a valid vague date for the observations.

The date, or date range, during which the sampling took place. 'Unknown' is an acceptable value. The date given here must be on or after the start date of the survey to which these observations are attached, and on or before the end date (if one was given).

Spatial Ref

Required, unless Location Name is the only location information.

Enter a spatial reference. If this box was empty and you selected a location, then it will default to the centroid for that location. You can edit this reference manually, or you can obtain a spatial reference from the map - see Getting grid references or locations from maps. If the point at which you clicked was within a boundary which is linked to a location, the name associated with that boundary will be returned in the location box - see Linking polygons to locations.

Spatial Ref qualifier

Required, if Spatial Ref. is entered.

See Spatial reference qualifiers.

Select a value from the drop-down box.

The value of the spatial reference qualifier is entered automatically in the following situations:

  • If the user has entered the spatial reference by using the Map browser, the value will be ‘Internal Map’.

  • If the user has selected a location, the value will be ‘Site Centroid’ by default.

Admin Area

Read-only. The information for admin area is displayed if there are associated admin areas for the location. To associate admin areas with a location, see Location Admin Areas.

Sample Type

Required.

Select a sample type from the drop-down box. The sample type is a ‘term-list’, and items in this list can be added or deleted by clicking Options - Term-lists. The icon for the sample type is used for the sample in the Observations screen.

Biotope

Optional. To enter a biotope, use one of the methods described in Find overview.

Recorder(s)

Required.

To enter a recorder(s):

  • Use one of the methods described in Find overview.

  • Click the [Add] button and the Find Individual dialog box is displayed. See Find dialog.

To delete a recorder:

  • Highlight the individual that you want to delete by clicking on the name.

  • Choose the [Delete] button.

Comments

Optional. Free text field for a comments about the sample.

Document

Optional. The recording card can be associated with an internal document. To enter a document, use one of the methods described in Find overview.

Species list grid

The species list grid is where you enter all of the information relating to the actual observations.

To record a taxon, click in the tick-box or select the row and press <space>. To remove the tick, ensure that all of the data for that taxon have been deleted, and then click in the box, or press <space> again. This method ensures that records are not accidentally deleted if they have associated data. If you add information in any of the columns for a given species, it will automatically be ticked.

To clear all of the observations that have been entered on a recording card without having to delete the associated data, click [Reset].

The number of species which have been ticked is shown in the species present label at the bottom of the recording card.

The columns of associated data that appear in the grid will depend upon how the recording card has been set up. Entering data in any of these columns is optional.

<F2> acts as the [Get] button in this grid.

Add Species

You can type the name or abbreviation of a species here. A search will be made for this species when you press <Enter>.

Recorder searches first to see if that species is within the species on the recording card.

If not; it searches depending on whether you have decided to restrict taxon searches to a rucksack, a checklist or to all preferred lists. By default; searches are limited to the preferred lists as these are most appropriate for current data entry.

If just one match is found, it will be added. If more than one match is found then the Find Taxon dialog will open to allow you to select the correct species.

  • If the selected species is already in the list displayed in the card, that species will be ticked and the list positioned so that it is visible.

  • If the selected species is not in the list, it will be added to the bottom of the list, ticked and then the list positioned to show it.

If you have added a Search Code for taxa to the rucksack used to provide the species list for the recording card, this code can be typed into the Add Species box to tick the required species.

If a delimiter, e.g. #, is entered for Rapid Entry Delimiter on the General tab in Tools - Options, data for all columns can be entered as a delimiter separated string in the Add Species box, e.g. Blackbird#2#3 <Enter> will tick Blackbird, place 2 in the column after the name and 3 in the next column. When you click in the Add Species box a pop-up appears showing the fields required with the next one to be entered in bold.

Sorting the list of species

  • The list will be, initially, in the order that the species were listed in the source list on which the Recording Card was based (i.e. a checklist or a list from a Rucksack).

  • Click on the headers "Scientific name" or "Common name" to sort in alphabetic order by that item. Clicking the header again will reverse the sort order (i.e. Z..A order).

  • To return to the original order, right click somewhere in the list and choose "Original order" from the pop-up menu which will appear.

Measurement columns

If measurement columns of the type Abundance of Qualifier (Unit), e.g. Abundance of None (DAFOR), are used, the values entered can be restricted to a set of allowed values by specifying the allowed values on the measurement unit in the term lists.

Saving the observations

To save the observations entered via a recording card, see Saving a Recording Card.

Modifying the recording card

If a species is no longer required on a recording card, right-click on it and select Remove Row from the pop-up box that appears.

Species added to the Species list grid using the Add Species box will remain in the grid until the recording card is closed.

When it is closed a message saying 'Do you want to save the changes to the list of taxa present on this card?' will appear. If you answer Yes, then the next time you use the card, the taxa added using the Add Species box will be listed in the Species list grid and taxa removed will no longer be listed. If you answer No, these changes will not be made to your recording card. Note that these changes are only made to the recording card, i.e. the .crd file that, by default, will be stored in the Recording Cards folder in the User Files folder in the Recorder folder, not to the rucksack, i.e. the .Ruk file, it was based on, if it was based on a rucksack. Where recording cards are stored can be changed using Tools – Options – File locations from the Main menu.

If the recording card is based on a rucksack the list of species in the Species list grid can also be modified by changing the rucksack, then using Data Entry - Enter Species for a place - Add or Edit Recording Card from the main menu to change the recording card. Add or Edit Recording Card can also be used to change the columns available in the Species list grid.

Tip: A quick way of entering records is to base a Recording card on an empty rucksack. This will give you a Recording Card with an empty species list. Then use the Add species prompt to type the name or abbreviation of each species you want to record and they will be added to the list.

Modifying data in samples

The data in a sample can be loaded into a recording card for editing. In some cases this will be easier than making the changes in the observation hierarchy itself.

To load data into a recording card:

  • Select a sample in the observation hierarchy and right click.

  • Select Load into Record Card from the menu that appears.

  • In the sub-menu, select the recording card that you want to use. If there are no cards listed, you will need to create a recording card (see creating a new Recording Card).

The Recording Card dialog box appears with the name of the recording card and sample in the title bar. As much of the data from the sample as the recording card can accommodate will be displayed and the taxa from the taxon occurrences will be ticked in the Species list grid. Taxa not on the recording card but for which there are observations in the sample will be added and ticked. If some of the data can't be displayed, because the recording card doesn't have a box or column for it, a warning will appear. Data that can't be modified will be displayed with a pale blue background. The species name displayed will be taken from the preferred determination. Data from other determinations won't be on view and hence can't be edited via this mechanism.

When sample data is loaded into a recording card, you can:

  • Edit the date, location, location name and spatial reference. Where appropriate, you will be offered the option to cascade the changes as specified in editing dates and location data. The other sample data will be read-only.

  • Add taxon occurrences by ticking additional species, or using the Add Species box.

  • Edit the data displayed for taxon occurrences. Note that the name can't be edited. If a name is incorrect untick the observation to delete it and add another one.

  • Delete taxon occurrences by unticking the species.

Once you have completed your changes, you can save them by clicking on [Save].

Note: Recorder's normal rules regarding custody and access levels apply when using this mechanism - see Editing an item.
