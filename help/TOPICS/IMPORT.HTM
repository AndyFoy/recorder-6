Import

Data can be imported into Recorder in a variety of formats. The most straight forward to handle is data exported from another copy of Recorder but observations can also be imported from various other file types, e.g. Excel, CSV (comma separated values) or text files.

To import data into Recorder , select Tools – Import Data from the Main menu.

You will be presented with the standard windows file selection box above from which you:

  • Locate the folder containing the file to be imported using the 'Look in' box.

  • Select the file type you require from the list.

  • Select the file that you want to import.

  • Click the [Open] button.

The next step will depend on the type of file you have chosen to import.

Importing data from another copy of Recorder

When data is exported from one copy of Recorder for importing into another copy it can be exported as an XML file or as a zipped database. If you are importing data from another copy of Recorder choose NBN Data (*.xml) or NBN Access Database (*.zip) depending on the extension (.xml or .zip) of the file you are importing. In this case, the import file will contain data in a format that can be added to your database without you having to provide any additional information so you will go directly to the analysis phase of the import.

Importing data from other sources

If you choose one of the other file types you will be guided through the Import Wizard as you will need to supply additional information to enable Recorder to import the observations.

 

Tip: If you want to know how much data you have imported in a particular import, use Help - Summary Info before and after an import and compare the statistics.

Tip: If you are in any doubt about an import back up your database before importing the data so that you can restore the back up if you are not happy with the result.

Tip: The file type ADO Connection enables you to import data from a Microsoft Access database.

Note: You may also see other file types which have been made available via Add-ins.
