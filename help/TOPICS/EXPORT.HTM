Export

To export data:

  • Select a Location, Name and Address, Survey, Survey tag, Survey event, Sample, Taxon observation or Biotope observation in the appropriate window.

  • Select Tools – Export Data from the Main menu.

OR

  • Use the Report Wizard to select observations.

  • Choose [Report Output] then Export Data on the Filter Result window.

Note: When using the Report Wizard to select observations for export, all data for the selected observations is exported not just the data that appears in the filter result.

Note: To export data for multiple surveys or selected taxonomic groups see Export management. To export data for multiple locations or names and addresses create a rucksack containing the required items and open it prior to selecting Tools - Export Data - see Export rucksack.

You will be presented with a dialog box:

Export type

Three options are available as standard:

  • NBN Data

  • NBN Access Database (zipped)
  • DMAP export

If Add-Ons have been installed in your copy of Recorder to allow you to do other types of exports, you will be able to select them using the drop-down arrow.

Note: NBN Access Database (zipped) is the preferred format for data transfer between copies of Recorder, with one exception. This export type produces smaller files that export and import more quickly.

When exporting data from Recorder 6 for importing into Recorder 2002 it is necessary to use NBN Data because this copes with differences in database structure between the two systems whereas the NBN Access Database (zipped) format does not.

Destination

Type the name of the file to which you want the exported information written.
The button opens a standard Windows Save As dialog to enable you to specify the file name and where you want it saved.

Reassign data custody

Clicking this option enables you to transfer ownership of your data to another Recorder system.

NOTE: You will not be able to edit data if you transfer ownership/custody of it to another Recorder system. See Transfer of ownership for details.

Include Observations

Only available when selecting locations and name and addresses for export. It allows you to choose whether to just export the selected item(s) themselves, or whether to also export all the observations linked to them.

Exporting locations and name and addresses without observations enables you to share these items with other Recorder users to save them having to enter them themselves and to avoid ending up with multiple versions of them when you exchange data.

Include Subsites for exported Locations

Only available when selecting locations for export. It allows you to choose whether to just export the selected location(s) themselves, or whether to also export all the subsites for them.

If you have chosen to reassign data custody the next screen will be:

  • Enter the Site ID/copy identifier of the copy of Recorder that is to be the new owner in "New Custodian key" - see Transfer of ownership for details of how to obtain this

  • Select the elements for which you wish to transfer ownership or "select all"

  • Click [OK] to perform the transfer of ownership. Click [Cancel] to abort the command.

Once the Export Type and Destination have been filled in, the [OK] button becomes active. Choose this button to perform the export. The [Cancel] button aborts the command without exporting any data. On the first occasion you perform an export, you will be asked to provide meta-data about the dataset - see Your first NBN export.

Note: Exports done by the methods described above will exclude:

  • Confidential observations unless a systems manager has ticked the Enable export of confidential occurrences option on the General tab in Tools - Options

  • Unchecked observations
  • Observations flagged as Failed/Pending Verification

Exports done using Tools - Export Management will include these observations although confidential observations will only be included if Enable export of confidential occurrences is ticked.

Note: when data is exported it is revalidated prior to export - for more information on validation, see Validating data. If invalid items are found a message saying 'There are invalid records in the data you are about to export. These records will not be exported. Do you wish to continue? If you say Yes the export will be done then, regardless of whether you said Yes or No, a filter will be loaded to enable you to see the invalid items.

A progress bar is shown in the status bar at the bottom of the screen during export.
