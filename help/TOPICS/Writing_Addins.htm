Writing Add-Ons

Add-Ons are Microsoft COM objects that implement particular interfaces with the Recorder application. They can be written using any Windows programming language that is capable of generating COM objects (Visual Basic, Delphi, C++, etc.). Add-ons are implemented as OCX or DLL files.

However, consider that:

  • Writing a COM object requires in-depth knowledge and experience of Windows programming.

  • Writing an Add-On requires the author to become familiar with the National Biodiversity Network data model, as implemented in Recorder, and in some cases with the object design of the Recorder application.

Technical documentation of the COM interfaces, the Recorder implementation of the NBN model, and the structure and nomenclature of objects within the Recorder application are available on the Recorder web site, www.jncc.gov.uk/recorder. Also available are the add-ins that can be installed in Recorder.
