Mapping- Pointer

When no other tool is selected, the map cursor is shown as a normal pointer . After you have used another tool, you can select the pointer by:

  • Selecting Map – Pointer from the Main menu.

  • Clicking the button on the active toolbar.

  • Pressing the short-cut key <F5>.

The pointer is used to:

  • Obtain a spatial reference by clicking on the map:

    • The co-ordinate of the point at which you clicked will be shown in the Spatial Ref. box at the bottom left-hand corner of the Mapping window, and will be formatted according to the currently selected spatial reference system. The point selected is at the tip of the arrow.

    • If this facility was accessed by pressing the [Get] button next to a prompt for a spatial reference, a small red marker will appear, temporarily, representing the spatial reference generated, along with a label containing the spatial reference.
    • If there is a polygon covering the point clicked and it is linked to a location in the location hierarchy, the location's preferred name will appear in the Location box at the bottom of the Mapping window.
    • See Getting grid references or locations from maps for more information

  • To select site polygons. Click on a line or polygon. It will change colour from the unselected colour to the selected colour to indicate it is selected - see Changing the settings or properties for a map layer to change these colours. If it is linked to a location, the location's preferred name will appear in the Location box at the bottom of the Mapping window.

  • To return to the cross hairs to draw a bounding box to define the area covered by a survey or the area to be used to retrieve data for a report.

If there are several overlapping site polygons at a particular point (e.g. sub-site boundaries within a site boundary), then repeatedly clicking will cycle through the polygons at that point. Depending on the patterns and colours used for your polygon layers it may be difficult to see which polygon is selected but, it should be possible to see that the boundary line has changed colour everywhere it is not directly overlaid by the larger polygon. Ideally, you should draw or import the largest polygon first and the smallest one last, then this problem will not arise.

NOTE: MapServer does not provide a means of 'bringing the selected polygon to the front', so it may be difficult to see that a smaller polygon overlaid by a larger one has been selected.
