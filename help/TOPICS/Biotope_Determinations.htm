Biotope Determinations

This page records the details of the determination(s) of the biotope occurrence. The name recorded here is taken from a particular classification. For field records, the determination is made by the recorder on the date of observation, and can really only be accepted or rejected. However, if some form of evidence exists which can be re-examined (photographs, field notes, etc), then later examination by the same, or another person may result in a re-determination. This could result in a different biotope name being applied to this observation or simply a confirmation that the original record was correct.

It is quite possible for this process to be repeated any number of times as knowledge and opinions change. In these circumstances, one determination must be selected as 'preferred' – i.e. represents the best available judgement as to the identity of the biotope in question. The name that will be used to display this observation on-screen and in reports is taken from the preferred determination.

Biotope

Required.

The name of the biotope taken from a particular classification. This is a reference to an entry in the Biotope Dictionary. It can be entered by:

  • Dragging & dropping, or copying & pasting a name from the Biotope Dictionary browser.

  • Typing part of a name and pressing <Enter>. This will display the Find Biotope name dialog showing matching names.

  • Choosing the button to display the Biotope Dictionary. Locate the desired name, and Return the data.

Note: if a second or subsequent determination is being added this defaults to the name from the preferred determination.

Determiner

Required.

The name of the person who identified this biotope. This is a reference to an entry in the Names and Addresses window. It can be entered by:

  • Dragging & dropping, or copying & pasting a name from the Names and Addresses window or a rucksack.

  • Typing part of a name and pressing <Enter>. This will display the Find determiner name dialog showing matching names.

  • Choosing the button to display the Names and Addresses window. Locate the desired name, and Return the data.

Note: only the names of individuals, NOT organisations can appear here. If you attempt to drag in the name of an organisation, find it in the dialog box or return an organisation's name from the Name and Addresses window, then that name will not appear.

Note: this defaults to the name of the person currently logged on, as displayed on the Welcome window, and will need to be changed if it is not the required name.

Role

Required.

Item selected from the determiner role term list. Users can add terms to this list if necessary. If this is a field record, the determiner's role will normally be 'Original Recorder', but for a re-determination, the determiner could be 'County Recorder' or 'Specialist'.

Note: determiner roles can be hidden so that they can't be used for data entry - see Determiner role for details.

Preferred

Flags the determination from which the biotope name that will be shown on screen, or in reports, should come. If there is only one determination, it will automatically be preferred. If there are several determinations, it will normally be the most recent or most authoritative. When entering data this defaults to being ticked.

Note: one, and only one, determination MUST be marked as preferred.

Type

Required.

Item selected from the determination type term list although it defaults to 'Unconfirmed'. Users can add terms to this list if necessary. Most observations will probably have no more information than this, but if the identification is subsequently re-assessed, then:

  • If the original identification is found to be correct, then this is a 'confirmation' and this field can be set to 'Correct' or 'Considered Correct'.

  • If the original identification is judged to be incorrect, but for whatever reason (explain in the comment) a correct identification cannot be attempted, then this can be set to 'Incorrect', 'Considered Incorrect' or 'Requires Confirmation'.

  • If the original identification is judged to be incorrect, but a new identification can be made, then this is a 're-determination', a different biotope name may be given and this field can be set to 'Correct' or 'Considered Correct'.

See Determination verification for details of how this is used to set the Verified flag on the biotope occurrence.

Note: determiner types can be hidden so that they can't be used for data entry - see Determiner type for details.

Date

Required, vague date.

The date of determination. Defaults to today's date.

Work

Can be used to record the documentary source that was used to make the identification. This is a reference to an entry in the Documents window. It can be entered by:

  • Dragging & dropping, or copying & pasting a name from the Documents window or a rucksack.

  • Typing part of a name and pressing <Enter>. This will display the Find Document name dialog showing matching names.

  • Choosing the button to display the Documents window. Locate the desired item, and Return the data.

Comments

RTF Memo. Any comment. This should be used when re-determinations result in name changes or rejections, to explain the problem.

To add a determination:

  • Click on the button, and the detail panel will go into Edit mode with some fields set to their defaults.

  • Type in the required information.
    • Click the button to save your changes.
    • Click the button to abandon your changes.

To delete a determination:

  • Highlight the determination that you want to delete.

  • Click the button.
  • You will be asked to confirm that you really want to delete this record.

To edit a determination:

  • Highlight the determination that you want to edit.

  • Click the button to put the detail panel into Edit mode.
  • Make the changes you require.
    • Click the button to save your changes.
    • Click the button to abandon your changes.

Note: you cannot change an item unless you have sufficient access rights and your copy of Recorder is the custodian of it - see Editing an item for more information, but you can add a new determination, if you have sufficient access rights, even though you are not the custodian.
