Edit and Browse modes

Recorder 2002 makes a distinction between 'Browse' mode, during which you are navigating around viewing data, but not changing anything; and 'Edit' mode, during which you are adding a new record or editing an existing one.

This is one of the most important aspects of the system for a new user to get to grips with, and remains a source of annoyance for experienced users! It is very easy to start typing some change to your data, only to find it is not accepted because you are not in Edit mode!

However, it was designed in this way to protect your data. The underlying database, Access 97, is 'normally' in Edit mode, and saves changes when you move to another record. If this behaviour was allowed in Recorder 2002, it would be very easy to change something unintentionally (by hitting the wrong key while moving from one record to another, for example). If you then move to the next record, the change is committed and the original data lost for good. Consequently, in Recorder 2002 you are forced to make an explicit decision to make a change – even though it can be irritating at times to have to remember to do so!

The detail panel on the right hand side of each of the main windows enters Edit mode when:

  • You press [Add] (to add a new record) or [Edit] to make changes to the currently selected record.

  • You select Edit – Add, or Edit – Edit from the Main menu.

How do you know that a window is in Edit mode?

  • The [Add], [Edit] and [Delete] buttons on the left are inactive.

  • The [Save] and [Cancel] buttons in the detail panel on the right are active.

  • Required fields have a coloured background, if you have selected this option (see Options – Appearance).

Once in Edit mode, to return to Browse mode:

  • You must ensure that all required fields contain valid data.

  • You can optionally enter data into other fields.

    • Choosing [Save] performs any outstanding validation checks and, if the data are valid, makes permanent changes to the database.

    • Choosing [Cancel] abandons any changes and returns the window to the state it was in before you entered Edit mode. No changes are made to the database.

Why can’t I get into Edit mode?

  • If the [Add] button is inactive, you do not have sufficient access rights to add data here.

  • If the [Edit] button is inactive, either you don’t have sufficient access rights to edit this record, or it is not your record and, therefore, you cannot change it. This happens if data have been imported from elsewhere and 'belong' to a different site.

  • If the [Delete] button is inactive, either you don’t have sufficient access rights to delete this record, or you cannot remove it because other data depend on it (e.g. a sample which has observations attached). You must delete the dependent data first

NOTE: Recorder 2002 does not allow cascading deletes. You must explicitly remove dependent records before you can remove the record they depend on.
