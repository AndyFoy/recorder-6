Additional Filters constraints

This page of the Additional Filters dialog allows the user to decide whether or not to include observations which are confidential, unchecked, flagged as Failed/Pending Verification or are negative records (i.e. taxon or biotope was looked for, but not found, and has therefore been recorded with a count or area of zero). All the options are selected (ticked) by default (i.e. exclude records which are confidential, unchecked, flagged as Failed/Pending Verification or have record quantity of zero, from the report).

 

Exclude Confidential Data

You will only be able to cancel this option (i.e. INCLUDE confidential records) if you have sufficient access rights - see Access levels.

Exclude Unchecked Data

If this option is unticked, observations which have not been marked as checked will be INCLUDED in your report.

Exclude Failed/Pending Verification Data

If this option is unticked, observations which are flagged as Failed/Pending Verification will be INCLUDED in your report.

Exclude Zero Abundance Data

If this option is unticked, negative records, i.e. those with Data = 0 (zero) on the Measurement tab on the taxon or biotope occurrence, indicating that they were looked for, but not found, will be INCLUDED in your report.
