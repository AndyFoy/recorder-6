Export in NBN Access Database format

NBN Assess Database (zipped) is a proprietary format in which data is written to a temporary Access database using the same table structures as the main database, then the resulting .mdb file is zipped. Because the data stays in Recorder’s table structures, there is little overhead in writing or loading data. Only user added data needs to be transferred because all copies of Recorder are assumed to contain the “system supplied” dictionaries and other supporting data, so the resulting file can be kept smaller. This is the preferred format for data transfer between copies of Recorder, with one exception. When exporting data from Recorder 6 for importing into Recorder 2002 it is necessary to use NBN Data because this copes with differences in database structure between the two systems whereas the NBN Access Database (zipped) format does not.

Advantages over the NBN Data format

  • Avoids the overhead of converting the data to XML and back again at the receiving end, therefore both export and import are quicker.

  • The resulting file is relatively small.

Disadvantages

  • Only meaningful to Recorder and therefore only useful for transfer between users of this package.
