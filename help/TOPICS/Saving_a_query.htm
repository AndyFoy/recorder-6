Saving a query

Queries produced by the Report Wizard can be saved to disk and reused in future using the Report – Run command. Normally, a query is saved together with a report layout or snapshot, so that running the report in future will do the whole job (retrieve the appropriate records from the database and lay them out as required). However, it is possible to save just a query, without linking it to a report template or snapshot.

Queries are saved from the Filter Results page of the Report Wizard. After you have designed a layout for the report and saved the template, you will come back to the Filter Result screen.

To save the query

  • Select File – Save as from the Main menu.

  • A standard Windows dialog will open, allowing you to give a name under which to save this query.

If you re-order columns in the Filter Result before saving the report, the change is remembered in the saved report.

TIP: Recorder uses long file names, and this is the name you will use to recognise each query. It is worth typing a suitably descriptive name so that you, and others, will recognise each one in the future.

Queries are saved with a .wzd extension. By default, they are stored in the Reports folder in the User Files folder in the Recorder folder, unless this is changed for the Queries entry using Tools – Options – File locations from the Main menu, but you can use the facilities in the Save As dialog to save them to any accessible folder.

Saving a query without an associated report layout

If you attempt to save a query before you have designed a report layout or defined a snapshot, you will see the following message:

  • Click [OK], and the Save dialog will be displayed, allowing you to go ahead with the save.

  • If you run this report using Reports – Run, you will be prompted to select a report template or snapshot at that time.

  • Click [Cancel], and the save operation will be abandoned.
