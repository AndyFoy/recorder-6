Import Wizard - data format

If you are using the Import Wizard to enter data into Recorder it will be worth putting some effort into preparing the data for import. With a little practice you will quickly identify changes you can make to speed up the use of the wizard. For example, some data types must have a value for every observation hence adding any missing values before starting the wizard will save time.

Alternatively, if people are sending you their observations in a spreadsheet, or other computer files, providing them with guidance on how to format the files can save a lot of time when you import their data. If you provide them with checklists of species names, location names and any other fields they want to use, and details of how to enter abundance data, you should find their data much faster to import.

For a list of the types of data you can include in each row, see the table of supported columns below.

The simplest format of data to be imported, and the easiest to deal with, is to use one row in the file for each observation and to divide it into a series of columns for the fields that need to be recorded: species name, location, spatial reference, observer, date, abundance, etc. Each row will become a Taxon Occurrence in the Observations hierarchy.

Spreadsheets naturally lend themselves to this format with:

  • Each row representing one observation

  • Each column containing one data type

Comma separated text files (.csv, .txt) are amongst the alternatives with:

  • Each line of the file representing a single observation

  • The fields of the observation separated by a special character (delimiter). A comma or a tab (ASCII 9) is often used, but you can use any character
  • Any data fields that contain a delimiter character MUST be enclosed in quotes. It is not uncommon to enclose all fields in quotes, or at least all the text fields.

Example:

"Syritta pipiens","West Wood, Close House","NZ1265","John Smith","12/08/2006"
"Syrphus ribesii", "Thornley Wood", NZ173604, "John Smith", "May 2005"

This data has five columns - species name, location, grid reference, observer and date. Note that the delimiter between the fields is a comma, but that the location field in the first row contains a comma. This is OK, because this whole field is enclosed in quotes.

General Information

Column headings: it is useful to put column names in the first row of data. This will help you remember what type of data the different columns contain and the Import Wizard may be able to use them to guess the type of data in each column from its name.

Ignoring columns: if the import file contains columns that you don't want to import you will be able select <skip this column> to ignore them.

Concatenating columns: if the import file contains more than one taxon occurrence comment column they could be concatenated so that they can be imported using a single comment column. In Excel, '&' can be used used to combine fields into one, e.g.

Supported columns

Preliminary Options

Location

Locations in this column will be matched to locations in your Location hierarchy. For any that don't match automatically you will be given a number of options on the location matching page of the import wizard. When a location is matched to a location in your location hierarchy, its name from the location hierarchy will be stored in (Survey) Event Location and Sample Location and the imported data will be stored in Event Location Name and Sample Location Name, if there is no Location Name for the observation in the import file. If Location and Location Name are present for an observation and Location remains unmatched the two will be concatenated and stored in Event Location Name and Sample Location Name. If Location is present and Location Name is not present, and Location remains unmatched the Location will be stored in Event Location Name and Sample Location Name.

If there is no location information in the import file, i.e. no Location, Location Name or Grid Reference columns, the wizard assumes all observations are for the same location and asks you to enter location data, as in Data Entry - Enter a species record, on the page where you enter other required data that is not in the import file.

If present, and Location Name is not present, a Grid Reference column must also be present and both must contain values for every observation.

See Location and Location Name for more information on the distinction between the two.

Location Name

Any text up to 100 characters long which supplements the information in Location or Grid Reference, if either of these two fields are present, but can be used on its own when further information is not available, e.g. for some historic data. If present on its own, i.e. without Location and Grid Reference, there must be a value for every observation. If present with Grid Reference but not Location, there must be a Grid Reference for every observation but Location Name is optional.

It is stored in (Survey) Event Location Name and Sample Location Name. See Location and Grid Reference for further details.

Grid Reference

Can be in any of the formats supported by Recorder including OS - GB, OS - Ireland, Lat/Long or UTM. This goes in (Survey) Event Spatial Ref. and/or Sample Spatial Ref. See Location, Location Name and Vice-County Number for further details. In most circumstances, if present, there must be a value for every observation.

Date

Can be in any of the formats supported by Recorder, i.e. a standard date or any of the vague date formats. The delimiter and order of elements in a date typed in all numeric format (e.g. 25/12/06) should follow the settings in your Windows configuration, although it can be altered via the formatting options on the template selection page of the wizard. If present, there must be a value for every observation. If not, the wizard assumes all observations are for the same date and asks you to enter a date on the page where you enter other required data that is not in the import file. This goes in (Survey) Event Date, Sample Date and Taxon Determination Date.

Species Name

Mandatory. Typically the scientific binomial, but it can optionally include the authority name. Alternatively, the common name can be used. The wizard will ask you to match these to names in the Taxon Dictionary.

Observer(s)

Text listing one or more observers (recorders) separated by , ; / \ "and" & +. If present, there must be a value for every observation and the wizard will ask you to match them to individuals in the Name and Address list. If not, the wizard assumes all observations are by the same observer and asks you to enter one on the page where you enter other required data that is not in the import file. These go in (Survey) Event Recorder(s) and Sample Recorder(s). The first observer becomes the determiner on the Taxon Determination if there is no determiner. Examples.

Determiner Name

Text containing a single determiner. If present, the wizard will ask you to match them to individuals in the Name and Address list. If not present, the determiner on the Taxon Determination is set to the first observer.

Abundance Data

A number of comma or semicolon (;) separated fields, e.g. 1 male, 2 females, 3 juveniles. If present, these go into the 'Data' and 'Qualifier' columns of the Measurements for the Taxon Occurrence. The wizard will ask you to match the qualifiers to Measurement Qualifiers from the Measurement Type term list items that are available on Taxon Occurrences. Can be used in combination with Taxon Occurrence Data. Examples.

Taxon Occurrence Comment

Any text. If present, this goes in the comment on the Taxon Occurrence.

<more options>

Abundance Accuracy

Text up to 20 characters long although the values on offer when entering data by other means are <None>, Exact, +/- 5% and Estimate. If present, there must also be a value in Abundance Data and it goes into the 'Accuracy' column of the Measurements for the Taxon Occurrence. If there is more than one set of Abundance data for an observation the value in this column will be applied to all of them.

Altitude

A valid number (positive or negative) with or without a suffix of 'm' representing altitude in metres. Values such as 10-20, >50 and <10 are also valid. If present, this goes in the Measurements for the Sample.

Associated Species

Species name for an associated species. If present, the wizard will ask you to match these to names in the Taxon Dictionary, and an association type is required. An associated species will appear as a Related Occurrence for the Taxon Occurrence.

Association Type

If present, there must be a value for each associated species, and the wizard will ask you to match these to the Relationship Type term list. If not, the wizard assumes all observations with an associated species have the same association type and asks you to select one.

BRC Source

Allows the record type to be specified using a standard BRC source value. If present, Record Type cannot be used, there must be a value for every observation and it must be one of the BRC source codes: FLD = field, in which case Record type will be set to 'field record'; MUS = museum or herbarium - Record type will be set to 'Collection'; LIT = Literature - Record type will be set to 'Reference'.

Biotope

Biotope code or description. If present, the wizard will ask you to match these to biotopes in the Biotope Dictionary. This will be used to create a Biotope Occurrence in the same Sample as the Taxon Occurrence.

Confidential

If present, then it can be 1, -1, Yes, True or Y for YES, or 0, No, False or N for NO. If YES then the Confidential flag will be ticked on the Taxon Occurrence.

Determination Reference

Text to be used to identify a reference. If present, the wizard will ask you to match these to documents in the Document list and the matched document goes in the Work field on the Taxon Determination.

Determination Type

If present, the wizard will ask you to match them to the Determination Type term list. If not, it will be set to "Unconfirmed". This goes in the Type field on the Taxon Determination.
Note that some determination types are hidden in some circumstances - see Determination Verification for details - but they are still available for matching in the import wizard.

Determiner Role

If present, the wizard will ask you to match them to the Determiner Role term list. If not, it will be set to "Original Recorder". This goes in the Role field on the Taxon Determination.
Note that some determiner roles are hidden in some circumstances but they are still available for matching in the import wizard.

MapMate Key

A globally unique key exported from the 'MapMate' biological recording software package. This consists of a string of 7 to 9 digits and lower-case letters. The last three characters identify the copy of MapMate from which the observations came (referred to by MapMate as the Centre Unique Key (cuk)). This will be translated to an NBN style, 16-character key in which the cuk becomes the Site ID, and will used as the key for the Taxon Occurrence. If present, Record ID and Site ID cannot be used.
Including this field will ensure that if the same observation is imported again, Recorder will flag it as a duplicate, and allow you to accept or reject it as specified in Duplicate Items, in the same way as for imports from other copies of Recorder.

Provenance

Text up to 16 characters long describing the provenance of the species, e.g. Alien, Native, Naturalised or Vagrant.

Publication Reference

Text to be used to identify a reference. If present, the wizard will ask you to match these to documents in the Document list and the matched document goes on the Sources tab on the Taxon Occurrence.

Record ID

An identifier for the observation supplied by the source system. This allows data to be imported more than once, e.g. if it is amended, without creating duplicate observations.
If present, Site ID must also be present and the MapMate Key must not be present.
The values must not exceed 6 characters in length and there must be a unique one for every observation.
When present, the primary key of observations created during the import process is constructed as follows:
Site ID + unique index + padding + Record ID.
The padding is a list of repeating zeros, bringing the total length of the key to 16 characters. The unique index is a 2 character ID which increments from 00 upwards in NBN fashion for each observation generated in a particular table using that combination of Record ID and Site ID. For example, if an imported observation creates 3 taxon occurrence data records, using Record ID 77 and Site ID TEST1234, then the keys used are:
1. TEST123400000077
2. TEST123401000077
3. TEST123402000077

Record Type

If present, BRC Source cannot be used, there must be a value for every observation and the wizard will ask you to match them to the Record Type term list. If not, the wizard assumes all observations have the same record type and asks you to enter one. This will appear as the record type of the Taxon Occurrence.

Sample Comment

Any text. If present, this goes in the comment on the Sample.

Sample Duration

Free text up to 20 characters long. Period over which sample was made e.g. 30 mins.

Sample Start Time

The time at which sampling commenced. This should be given using 24hr clock.

Sampling Method

If present, there must be a value for every observation and the wizard will ask you to match them to the Sample Type term list. If not, the wizard assumes all observations have the same sampling method and asks you to enter one. This goes in the Type field in the Sample.

Site ID

An 8-character NBN globally unique identifier for the system from which the observations originated. This allows the source to specify the Site ID to use as the custodian for all observations created by the import process, excluding those created using the Make New Entry option. This is also used to combine with the Record ID to create a valid key for the Recorder database.
If present, Record ID must also be present, the MapMate Key must not be present, and there must be one for every observation.

Note: It is important that Site IDs identifying different systems remain unique. If they don't there is a danger that data from one system will overwrite data from another when imported into Recorder. Please use this field with caution and check with JNCC (e-mail: Recorder@JNCC.gov.uk) that the Site ID you plan to use is unique.

Note: Site ID and Record ID must ONLY be used for data coming from systems other than Recorder and MapMate. They must NOT be used to import data from another Recorder system because of the potential for corruption as explained below.

Spatial Reference System

A code up to 4 characters long for the Spatial Reference System of the Grid Reference. In a standard copy of Recorder these will be:
OSGB: Ordnance Survey - GB Grid
OSNI: Ordnance Survey - Irish Grid
LTLN: Latitude and Longitude
UTM: UTM (Universal Transverse Mercator co-ordinates)
Others will be available if add-ins have been installed to make them available.
The system currently in use is the one ticked on Tools - Options - Spatial References.

Specimen Comment

Any text. If present, Specimen Number and Specimen Type must also be present. This goes on the Specimens tab on the Taxon Occurrence.

Specimen Number

An identifier up to 30 characters long which could be an accession or registration number. If present, Specimen Type must also be present. This goes on the Specimens tab on the Taxon Occurrence.

Specimen Type

If present, Specimen Number must also be present and the wizard will ask you to match them to the Specimen Type term list. This goes on the Specimens tab on the Taxon Occurrence.

Substrate

If present, the wizard will ask you to match them to the Substrate term list.

Survey Event Comment

Any text. If present, this goes in the comment on the Survey Event.

Survey Event Weather

Any text up to 200 characters long. If present, this goes in Weather on the Survey Event.

Surveyor's Ref

Any text up to 30 characters long. If present, this goes in the Surveyor's Ref field of the Taxon Occurrence.

Taxon Determination Comment

Any text. If present, this goes in the comment on the Taxon Determination.

Taxon Occurrence Data

When this column type is selected, you will be able to select any number of column types from every possible combination of Measurement Qualifier and Measurement Unit from the Measurement Type term list, provided they are available on taxon occurrences. If present, these go into the Measurements for the Taxon Occurrence where the 'Qualifier' is determined by the column type and the 'Data' is the contents of the column in the import file. Can be used in combination with Abundance Data. Example.

Vice-County Number

A valid Watsonian vice-county number, or an Irish vice-county number prefixed by 'H' - e.g. H6, Waterford. If present, and Location is not matched to locations in the location hierarchy, the observations will be linked to the vice-counties which are sub-sites of 'Irish Vice-counties' or 'Vice-counties' in the Location hierarchy. When present, Grid Reference must also be present in the input data and both must have a value for every observation.

Note: The supported columns above are available in Recorder 6 up to and including version 6.16. However, the Import Wizard has been designed in such a way as to make it comparatively easy to modify it to import additional fields so subsequent versions may support additional column types. Check what is currently available by selecting <more options> in the drop down box for selecting column types.

To consult other users about whether your requirements are likely to be of more general use, please make use of the Recorder Forum, part of the NBN Forum, which can also be accessed from this web-site.

Site ID and Record ID

When users have received Recorder data in an Excel file rather than as a Recorder export, some of them have contemplated importing it using the Site ID and Record ID column types in the import wizard, the idea being to split the taxon occurrence key into Site ID and Record ID. The Site ID and Record ID were provided to allow data to be imported from other databases using identifiers from the system concerned so that if corrections to existing observations are imported they will overwrite the original version rather than create new versions.

Site ID and Record ID must NOT be used for data from another copy of Recorder (Recorder 2000/2002 or Recorder 6) because of the potential for corruption.

If you use the import wizard to import a record with say, Site ID SR000247 and Record ID 004444, the survey event, sample and taxon occurrence will all have keys of SR00024700004444, unless there is already a suitable survey event and sample. The problem is that the survey event SR00024700004444 may exist in the original system but will probably be linked to different observations due to the way keys are created when data is entered via the standard means. Likewise with the sample. Hence, if this data was imported into a system containing the survey event/sample SR00024700004444 created by standard Recorder data entry one version would overwrite the other probably leaving some taxon occurrences linked to the wrong samples.

The correct way to handle this situation is to go back to the holder of the data and get them to do a standard Recorder export in NBN Data (.xml) or, preferably, NBN Access Database (zipped) format and import that. If you don’t know who the owner is JNCC will be able to tell you who it is, provided the Site ID has been registered. If it hasn’t, they will, at least, be able to tell you who the reseller was, and they should be able to tell you who the owner is, subject to data protection issues.
