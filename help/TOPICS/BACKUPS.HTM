Making backups

It is wise to make regular backups of your data, and also to make a backup before and after you make any major changes, such as importing a quantity of data.

Two major elements of your system need to be backed up, the SQL Server database and the application files.

Backing up the application files

In a standard standalone installation with Windows XP, the application files will typically be in a folder called Recorder 6 in C:\Program Files, although other options can be specified during installation.

In a standard network installation with Windows XP, the application files will typically be split between a folder called Recorder 6 Server on a server drive and one called Recorder 6 Workstation on each computer that acts as a workstation, although other options can be specified when installing the system.

The location of some of the application files, e.g. rucksacks, recording cards, saved reports generated by the Report Wizard, xml reports, saved templates (report layouts) and xml batch updates, will be the folders listed in Tools – Options – File Locations.

Note: By default, these folders will not contain your SQL Server database - see below - although they will contain the linked Access database: \Database\NBNData.mdb.

These folder(s), or the files they contain, should be backed up after significant changes to them, but they are unlikely to need backing up as often as the SQL Server database. They can be backed up by copying the files or the entire folders to a CD, DVD, external hard drive or network drive. You may choose to compress them with the compression facilities in Windows or software such as WinZip. Compressing the folders will greatly reduce the amount of space occupied by the backed-up copies of them.

See the installation guides for more information about the location of the application files. There are standalone and network installation guides in the Documentation folder on all the Recorder 6 installation DVDs and in the Resources section of the Recorder web site, www.jncc.gov.uk/recorder.

Note: Recorder 6 version 6.17 is officially supported for use with Microsoft Vista and Windows 7 but the installation guides have not yet been updated. Until they are, for more information, please consult the Recorder Forum, part of the NBN Forum, or contact the supplier of your copy of Recorder, or any of the Recorder suppliers who are listed on the Recorder web site, www.jncc.gov.uk/recorder.

Backing up the database

The data you enter into your copy of Recorder 6 will be stored in the SQL Server database. It is most important that this is backed up regularly. If your system crashes, the amount of data you lose will depend on when you last backed up your database and whether it has been stored on a medium that is not affected by the crash.

For a Recorder 6 installation, the database will be a file called NBNData_Data.MDF. Associated with it will be a log file called nbndata_data_log.LDF. In a standard standalone SQL Server Express installation with Windows XP, these will be stored in the Data folder in a Microsoft SQL Server folder, e.g. C:\Program Files\Microsoft SQL Server\MSSQL.1\MSSQL\Data for SQL Server 2005 Express. For SQL Server 2008 Express, the \MSSQL.1 will be replaced with the name of the appropriate folder in its installation. If you have SQL Server tools such as Enterprise Manager (SQL Server 2000) or SQL Server Management Studio Express (SQL Server 2005/2008 Express) you will be able to use them for backing up your database. Alternatively you can use the facilities in Recorder 6.

Note: By default, in a standard standalone installation with Windows XP, a backup done in Recorder 6 will be a file called NBNBackup.bak in C:\Program Files\Microsoft SQL Server\MSSQL.1\MSSQL\Backup for SQL Server 2005 Express. In older installations it may be in the Temp folder of the user who installed Recorder 6, e.g. C:\Documents and Settings\User Name\Local Settings\Temp.

Recorder 6 allows users with system administrator access to view and change the location where this backup is placed.

Viewing or changing up your backup location

  • Open Recorder

  • Select Tools – Database Tools – View or Change Backup Location

  • Note the position of the backup

  • Click the [Cancel] button if just viewing the location

  • If changing the location, enter the required location and click the [OK] button

Making a backup

  • Open Recorder

  • Select Tools – Database Tools – Backup Database

  • Wait for the database to backup

  • Copy the backup file, NBNBackup.bak, to somewhere safe, e.g. to a CD, DVD, external hard drive or network drive, so that it is available to restore from.

You may choose to compress this file with the compression facilities in Windows or software such as WinZip. Compressing it will greatly reduce the amount of space occupied by the backup. It will not fit on a standard CD unless it is compressed. Note that the compression facilities in some versions of Windows will only compress things up to 2 GB in size.

Note: Each time a backup is done in this manner NBNBackup.bak will be overwritten, so if it is to be retained a copy must be taken.

Restoring from a backup

  • Place the NBNBackup.bak file to be restored in the backup folder, i.e. the folder mentioned in the Note above unless it has been changed

  • Open Recorder

  • Select Tools – Database Tools – Restore Database
