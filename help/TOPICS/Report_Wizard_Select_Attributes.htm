Select Attributes

You see this screen when you have finished specifying the criteria for the observations you wish to retrieve. It asks you to choose which data items you want to print or export in the final report. The items from which you can choose depend upon the options you chose in earlier screens.

  • The fields are arranged in separate groups according to where they appear in the observation hierarchy. Click the to view the options in each group. Choose the data fields you wish to print or export by clicking the box next to the label with your mouse - click again to de-select an item. Selected items are ticked.

  • You can also move the highlight bar up and down the list using the <up arrow>, <down arrow>, <PgUp>, <PgDn>, <Home> or <End> keys, and press the spacebar to select and de-select items.

  • The fields ticked in the screen shot above are the ones required to include the date, recorders (observers) and location data for observations as input using Data Entry – Enter a species record or Data Entry – Enter Species for a place. The date and location data could also be selected from the (Survey) Event although the sample data can be more detailed than the event data and so should be selected if the more detailed data is required, e.g. an event for a specific location may have the central spatial reference of the location in Event Spatial Reference but it may contain multiple samples each with different Sample Spatial References for different parts of the location.

  • Multi-value fields like Sample Recorders, Sample Administrative Areas and Sample Location Administrative Areas will be output as comma or semi-colon separated strings.

  • To include measurement data for observations like abundance, select the required measurement type and qualifier combinations from the Measurements group within the Taxon group. Alternatively, select Obs Abundances (LC) from the Observation group, which will place all the measurements for observations in a single column as a semi-colon separated string.

  • To include designations, click the beside the required designation set to view the options and select the required field(s) from Short Names, Long Names, Kind, Status Abbreviation and Taxon Designated (Yes/No). If there is no Status Abbreviation available, Short Name will be used instead when this is selected. To view what will be included in the report for these options, select Tools - Term Lists from the main menu and select the Taxon Designation Type term list. The designations for an observation will appear as a single string with the individual designations separated by the list separator for your system, usually comma or semi-colon. This is a Microsoft Windows setting that can be changed via Start - Control Panel - Regional and Language Options.

The Selected Designations set is for use to include the designations in a designation set or sets that you selected via the Designation Selection page of the wizard, the page you see if you selected the Restrict report to one or more Designations option in the starting screen of the Wizard.

  • If you want the report in taxonomic order select Recommended Taxon Sort Order and sort on that in the next window of the wizard. This will work correctly for all taxa that are linked through properly in NameServer. Recommended Taxon Sort Order works across lists in the taxon dictionary whereas Taxon Sort Order only works if all the observations in the report have been entered using the same list in the taxon dictionary.
