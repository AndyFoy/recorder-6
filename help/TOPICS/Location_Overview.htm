Location overview

One of the fundamental pieces of information about a biological observation is the ‘place’ where the observation was made. In Recorder there are two ideas for ‘place’.

The first is a ‘location’, which is a named area of land (or sea) to which additional information can be attached (such as county, ownership, management etc.). The second is a ‘spatial reference’. This consists of a simple grid reference or latitude and longitude where the observation was made. You can also store some text describing the point in the Location name field which appears in Survey events and Samples.

The Locations window is where additional information is managed for a named location. It forms a fundamental screen in Recorder . In addition, it is also used to hold information on Location features.

The overall layout of the screen is the same as the other screens in Recorder , i.e. a tree view in the left panel, and a detail window in the right panel which displays information about the selected item. The tree view on the left displays all of the locations in the database as sites or sub-sites, and these can be arranged in hierarchies.
