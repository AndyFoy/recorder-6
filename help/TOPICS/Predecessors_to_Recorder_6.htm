Predecessors to Recorder 6

 

Recorder 3

Microsoft DOS-based software developed by Dr Stuart Ball of the JNCC c. 1993. Recorder 3 is a relational database application written in Advanced Revelation (ARev). It was used by many individual recorders and organisations, including local record centres, across the UK. The system is still in use, but it is no longer formally supported. Recorder 3 has been superseded by Recorder 2000/2002 and, more recently, Recorder 6.

Revelation Software withdrew all support for any ARev for DOS application runing on Microsoft Vista machines in 2007. ARev has been superseded by ARev32, which forms part of OpenInsight, and Recorder 3 would need to be upgraded to ARev32 to run with Vista.

 

Recorder 2000

Microsoft Windows-based software developed by Dorset Software on behalf of JNCC. Recorder 2000 was released in September 2000 as a replacement for the DOS based Recorder 3. It is a relational database application which holds all its data in a Microsoft Access 97 database based on the NBN Data Model.

Recorder 2000 can be installed on a standalone PC or on a network.

 

Recorder 2002

Recorder 2002 was a major upgrade to Recorder 2000, hence the change of name. A preliminary release of Recorder 2002 - version 2.0.5 - came out in December 2002 and the installation CD for this version was commonly referred to as the 'white' CD. It was followed by version 2.3.1 in December 2003 for which a new installation CD was produced, the coloured one with a picture of a bee nectaring on a cowslip on it. The latest version of Recorder 2002 is version 2.3.7.8.

To install Recorder 2002 you need to:

  • Install Recorder 2002 using the version 2.3.1 CD. If you don't have a copy of this CD, anyone who purchased earlier versions of the system can obtain a copy free of charge from John Morris at JNCC, e-mail John.Morris@jncc.gov.uk.

  • Upgrade to version 2.3.6.5, released in July 2004.

  • Upgrade to version 2.3.7.8, released in March 2005.

The upgrades are available from the Recorder 2002 section of the Recorder web site. Depending on the power of your PC and the speed of your network, where appropriate, upgrading to version 2.3.6.5 may take upwards of one hour because it is adding a lot of information to the taxon dictionary, including 12 new checklists. The upgrade to version 2.3.7.8 adds another 11 checklists to the taxon dictionary but it does not take nearly as long.

Please refer to the associated release notes for details of the changes in a particular release and for additional instructions, e.g. after upgrading to version 2.3.6.5 and to 2.3.7.8 it is necessary to rebuild the taxon indexes.

The size of a Microsoft Access 97 database, and hence the Recorder 2002 database, is limited to 1 GB and JNCC have estimated that this means the capacity Recorder 2002 is approximately one million observations. The biological recording community needs a system with greater capacity so JNCC have developed Recorder 6, a SQL Server system. In a licensed installation of SQL Server the size of the database is more or less unlimited. It will only be constrained by the capacity of the hardware available.
