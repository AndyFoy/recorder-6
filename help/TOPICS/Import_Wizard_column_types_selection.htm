Import Wizard - column selection

The column selection page of the import wizard enables you to specify the type of data in each column in the import file. To do this:

  • Click on the drop-down arrow in the Type row for a column.

  • Select the appropriate type for the data in the column. The most commonly used types are listed initially. If the type you require is not listed, click <more options> to see the full list of available column types - see the table of supported columns in Data Format for details.

  • If you don't want to import the data in a particular column, select <skip this column>.

If the first row of the data to be imported contains column names, tick the box next to that option. The Import Wizard will try to guess the types of columns from their names. You can still edit the types of any columns that the Wizard guesses incorrectly. The number of observations being imported is displayed in the bottom left corner.

Note: You need to select a Type for all the columns to be imported. Any columns with a type of <skip this column> will be ignored.

If you chose to use a template saved after a previous import, the column types that were saved will be re-applied provided that the column names are still the same.

When you have completed specifying the type of data in each column click [Validate Columns]. If errors are found a count of them will appear at the top, the rows containing the errors will be shaded pink and the invalid data will be shaded red. You won't be able to proceed, i.e. the [Next] button won't become active, until all the errors are cleared. If you float the cursor over a red box, a message will appear giving some information about the error.

In the grid you can:

  • Edit the invalid data by clicking in a particular cell and typing a new value. Once you move away from that cell, the change will be saved. Until that point, pressing <Esc> will restore the cell to its original value.

  • Select a single row by clicking the row number in the left most column.

  • Remove selected rows using the [Remove Selected Rows] button at the top.

  • Filter on selected rows by ticking the "Show selected rows" box at the top.

  • Filter on rows containing errors by ticking the "Show rows with errors" box at the top. When this filter is on a row will disappear from the grid when all the errors in it are cleared.

  • Sort by the values in a particular column by clicking the header, i.e. by clicking in one of the shaded boxes in the row below the Type row. Clicking once sorts them into ascending sequence, clicking again sorts them into descending sequence and clicking in the box below "Type" returns them to their original sequence, row number sequence.

To select multiple consecutive rows, select the first one then hold down the <Shift> key and select the last one.
To select multiple rows that aren't consecutive, select the first one then hold down the <Ctrl> key and select the others in turn.

Correcting Problems

Column Types

The only mandatory column is Species Name but there are rules for certain other column types, e.g.

  • For Observer(s): "If present, there must be a value for every observation and the wizard will ask you to match them to individuals in the Name and Address list. If not, the wizard assumes all observations are by the same observer and asks you to enter one on the page where you enter other required data that is not in the import file."

  • For Specimen Number: "If present, Specimen Type must also be present."

See the table of supported columns in Data Format for details.

If the data being imported contravenes any of these rules, information to help you resolve the problem will appear at the end of the text on the left, along with other tips on what needs to be done before you can proceed. The [Next] button won't become active until all the issues are resolved.

Errors

When you click [Validate Columns] Recorder validates the data in each column according to its type. All errors have to be cleared before you can proceed. As you clear the errors, the number at the top will decrease. The errors illustrated above are typical of the types of errors found at this stage of the wizard, i.e.

  • Invalid dates:

    • Pre 2006 is not a valid Recorder date, i.e. it isn't a standard date or a date in any of the vague date formats supported by Recorder. It should be changed to "-2005".
    • 31/06/2006 and 1/17/2006 should perhaps be changed to 30/06/2006 and 1/11/2006 respectively.

  • Invalid grid references: GB grid references must be two letters followed by an even number of numeric digits. Irish grid references must be one letter followed by an even number of numeric digits.

  • A location must be accompanied by a grid reference.

  • A number of fields are a fixed number of characters in the Recorder database, e.g. Location and Location Name - see the list of supported columns in Import Wizard - data format for details. When the data being imported exceeds the number of characters that can be stored in a field it will be flagged as an error so that the you can abbreviate it appropriately rather than have the system truncate it without warning. This doesn't apply to many of the comments as they are "Memo" fields which means they can contain large amounts of data.

Note: When you have cleared all the errors you will need to validate the data again using the [Validate Columns] button.

Other types of errors will be detected during the Analysis phase, e.g.

  • Dates that are outside the date range allowed for the survey to which the observations are to be imported.

  • Grid references that are outside the bounding box for the survey to which the observations are to be imported.

  • Grid references that are outside the grid squares for the locations to which the observations are to be linked.

You may like to check for these types of errors, particularly for dates being outside the range allowed for the survey, before attempting to import a file.

Note: Correcting errors and removing records in this screen does not make any changes to the original data file. The changes are only to the copy of that file taken at the beginning of the import process.

Once all issues have been resolved, the [Next] button will become active. Click on the [Next] button to move to the next screen.
