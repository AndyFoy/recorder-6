Survey Tags

Surveys in the Observation hierarchy can be organised into folders by assigning one or more tags to them. The folders appear as the top level in the hierarchy and the folder names are the tags, as illustrated below. This is particularly useful for users who have numerous surveys and/or for those who import a lot of data. This facility becomes available when Organise Surveys by Tag is ticked on the General tab in Tools - Options. If this is not ticked, the surveys appear in a list in alphabetic order at the top level in the hierarchy - no folders are on view.

The tags can be created and assigned to surveys in the observation hierarchy. They can also be created via Tools - Enhanced Termlists.

Survey Tags can be used to determine the surveys included in reports, and those included in exports.

The screen shot above shows the properties of a tag, including the surveys assigned to it. The one below shows the tags tab on a survey. These windows are accessed by selecting Data Entry - Observations from the Main menu.

Note: a survey can have multiple tags. Although this means it appears in multiple folders, the data is not duplicated in the database.

Note: surveys that have no tags will appear after the tags at the top level in the hierarchy.

To add a new Survey Tag:

  • Select an existing survey tag or survey, if present, then either:

    • Select Edit – Add from the Main menu.

    • Choose [Add] at the bottom of the left-hand list.

  • Select Survey Tag. A <New Survey Tag> indicator will appear.

  • Enter a label for the tag, and a description, if required.

  • To enter surveys use one of the methods described in Find overview.

  • Choose [Save] in the right-hand panel of the window to update the database with the new item.

  • You can choose [Cancel] in the right-hand panel to abandon any changes. No changes will be made to the database, and the window will be restored to its original state.

To edit an existing Survey Tag:

  • Ensure that the item you wish to edit is highlighted in the list on the left of the window, then either:

    • Select Edit – Edit from the Main menu.

    • Choose [Edit] at the bottom of the left-hand list.

  • When you have made the required changes, choose [Save] in the right-hand panel of the window to update the database and make the changes permanent.

  • You can choose [Cancel] in the right-hand panel to abandon any changes. No changes will be made to the database, and the window will be restored to its original state.

To remove a Survey Tag:

Remove the survey tag from all associated surveys:

  • Ensure that the item you wish to remove is highlighted in the list in the left of the observation hierarchy window, then either:

    • Select Edit – Delete from the Main menu.

    • Choose [Delete] at the bottom of the left-hand list.

  • You will be prompted to confirm that you really want to clear the survey tag from all associated surveys.

  • Select Yes or No as required.

Remove the survey tag:

  • Select Tools – Enhanced Termlists from the Main menu.

  • Select the tag to be deleted so that it is highlighted in the list on the left of the window, then either

    • Select Edit – Delete from the Main menu.

    • Choose [Delete] at the bottom of the left-hand list.

  • You will be prompted to confirm that you really want to delete the selected term and all its synonyms.

  • Select Yes or No as required.

Note: dragging and dropping a survey copies it, Ctrl+dragging a survey moves it. Copy does not clone the survey; it means the original survey has both tags, and Move means the tag folder that the survey is moved from is removed and the tag folder that it is dropped to is added.

Note: survey tags are NOT exported as part of a survey - they are a local organisational tool only.
