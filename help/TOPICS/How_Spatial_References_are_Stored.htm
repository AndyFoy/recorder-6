How spatial references are stored

Internally, spatial references are converted to latitude and longitude co-ordinates in degrees, and are stored as a pair of floating point numbers. However, the original string that was entered by the user is also stored. This allows information about the precision of the original co-ordinate to be derived. It can also be used to check for rounding errors that might arise if co-ordinates were repeatedly converted between co-ordinate systems.

Five fields are always stored:

Spatial_ref

The string entered by the user (this is 'cleaned up' before storage – e.g. letters are forced into upper case, extra spaces are trimmed).

Lat

Latitude in degrees.

Long

Longitude in degrees.

Spatial_ref_system

This identifies the system that is used to covert to and from the value that was entered by the user to the stored lat/long (e.g. OSGB).

Spatial_ref_qualifier

Information obtained from the user about how the spatial reference was obtained (e.g. 'estimated from map').

If a table stores more than one spatial reference, these field names are prefixed with a label indicating their purpose.
