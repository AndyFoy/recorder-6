Submitting observations to a scheme or organisation

If you regularly submit observations by email, this facility allows you to automate much of the process by setting up a Recording Scheme. Observations can then be submitted via e-mail at a touch of a button. Recording schemes are usually based on an export filter. If a Recording scheme is not based on an export filter, it will export all of the observations from your dataset.

Before you set up a Recording scheme, you will probably want to define the observations you wish to include by creating an export filter.

To set up the details of a scheme or organisation:

  • Select Tools - Export Management - Manage Recording Schemes from the Main menu.

  • Just fill in the cells of the grid to make an entry.

    • Press the <Insert> key to add a blank line for a new entry.

    • Press the <Delete> key to delete the selected row.

  • Press [OK] to save your changes or [Cancel] to close the dialog without making any changes.

Recording Scheme

Enter a title for the scheme or organisation. This is free text and is just used to identify this entry in the menu.

EMail

The email address to which the observations should be sent.

Filter Name

You can, optionally, select an export filter from which the observations will come. If you leave this blank, then all new and updated observations in the whole database will be submitted.

Export Format

Choose whether to send an NBN Data (.xml) file or an NBN Access Database (zipped). The latter is appropriate if you know that the person or organisation to whom you are contributing is also using Recorder to manage the data.

Last Contribution Date

This is recorded automatically and you cannot edit it. It will initially be blank until after the first time you submit to a new scheme.

Save

This allows the selected entry to be saved to a file. This file could then be distributed to other Recorder users.

Open

Allows a file, saved by the Save facility, to be read. This is intended to allow an organisation or scheme to supply its details to Recorder users in order to facilitate the observation submission process.

To submit observations:

  • Select Tools - Export Management - Contribute to Scheme - <scheme name> from the Main menu.

  • Recorder will automatically select observations, export them, zip the resulting file and send an email with the zip file attached. The Last Contribution Date in the Recording Scheme Manager will be automatically updated to the current time and date.

  • At the end of the process, your email software will be opened and a message will be inserted into it with the export file attached. Your contribution is ready to send, so all you need to do is press the emailer's [Send] button. However, you can edit the message text to add a cover note before you actually send the email if you wish.

NOTE: Recorder uses MAPI to send the email. This will only work if MAPI compliant email software is installed on your machine. If such software is not installed then email access will not be available (and you will see a message informing you of this). However, the zipped, export file will be still generated. You can copy this file to a floppy disk and post it to the scheme or organisation, or attach it to an email you send using your non-MAPI software.

The export file

  • The file which is generated by this process is called "NBNRecords_<SiteID>_<date>.zip" (e.g. NBNRecords_JNCCDEV1_20020701.zip). It contains a file called NBNRecords.XML or NBNRecords.mdb depending on which export format option you selected.

  • It is saved in your Windows temporary directory. The location of this directory depends on the version of Windows you are using (but can also be configured by the user). Generally it will be in "C:\temp" or "C:\Windows\Temp" in earlier versions (e.g. Win95) or "C:\Documents and Settings\<user name>\Local Settings\Temp" in the later versions (Win2000/XP).
