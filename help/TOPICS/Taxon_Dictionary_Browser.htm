Taxon Dictionary Browser

The Taxon Dictionary browser displays information on taxonomic checklists (see Dictionaries overview) in a screen with two panels. These data are Read-only, and cannot be edited – the browser is simply for displaying the data, and returning or dragging items to other screens. To add data to the Taxon Dictionary, see New Taxa General.

The Taxon browser consists of two panels, and a drop-down box for the checklist name:

  • To display a checklist, select a name from the drop-down box.

  • The left hand panel displays the scientific names in the checklist either as a hierarchy or as a simple 'flat' list.

  • The right hand panel shows the detail of the item which is currently selected in the left hand panel (see Taxon Browser Detail). This pane can be turned on and off (see Hiding and showing dictionary details).

Note: The left hand panel will display common names as well as the scientific names if 'Use Common Names' is ticked on the General tab in Tools - Options.

To find a taxon in the Taxon Dictionary, see Finding Taxa.

Note: In certain circumstances searching the dictionary will be noticeably faster if the dictionary details are hidden.

The Preferred lists

To aid users, those checklists that are most useful for entering current observations have been grouped together at the beginning of the drop down list. The names of these lists are preceded by the name, in CAPITAL LETTERS, of the taxonomic group they cover. They are the 'Preferred' lists. When entering data for multiple taxonomic groups, users can simplify, and hence speed up, data entry by restricting searches for taxa to just these lists by setting 'Restrict Taxon Searches to:' 'Preferred Checklists' on the General tab in Tools - Options or on the Find Taxon dialogue box.

If in doubt about which list to use in the dictionary, you are advised to set Restrict Taxon Searches to 'Preferred Checklists' and allow Recorder to select taxa from the appropriate list. Alternatively, users could use the Recorder 3.3 (1998) checklist as it includes most taxa found in the field and is therefore a good general purpose list for data entry. However, as the date indicates it is now rather old and it may not contain all the taxa you require, in which case it will be necessary to select from another list. For example, if you are recording fungi you may find that the species for which you want to record observations are not on the Recorder 3.3 (1998) checklist. In this case you should use the FUNGI&LICHEN British Mycological Society GB Checklist of Fungal Names, the preferred list for fungi and lichen.

Status and legislative lists

Status and legislative lists, e.g. Priority Species List (2007), Red Data Book and Wildlife and Countryside Act lists, are included in the Taxon Dictionary for reporting purposes. These lists cannot be used for data entry. If you try to select a taxon from one of them when entering data you will receive a message saying 'The taxon selected is from a species list that is inappropriate for data entry purposes. Please choose a taxonomic checklist to select the taxon from.'

Using these lists, a report can be produced, for example, for all taxa on the Priority Species List (2007), i.e. for the UKBAP species. If required it can be restricted to various dates, locations, recorders etc.

Other lists

A number of other lists have been included in the Taxon Dictionary for historic reasons or because they are needed for data entry but are not classed as preferred lists. Some of them are exact replications of published checklists; they are not edited and do not necessarily contain a hierarchy or common names, unless in the published list. Long lists that are not arranged in a hierarchy are cumbersome to use as they take significantly longer to open than other lists.
