System supplied vs User added

Information supplied with Recorder, mainly as part of the dictionaries or term-lists, is marked as 'System-supplied'. Updates (e.g. of dictionaries) supplied via the NBN web site will be similarly marked (see the NBN website).

Users are not allowed to edit information which is marked as system-supplied.

When users make additions to term-lists or add new taxa or biotopes, these entries are distinguished from the centrally supplied and supported entries because they are not marked as system-supplied.

NOTE: New taxa and biotopes that are added by the user, and are therefore not marked as system-supplied, are not exported (which means none of the data associated with them, such as observations, are exported).

To export such data:

    • Get the taxon or biotope added to the centrally supported dictionary via an update (see how to get taxa and biotopes added).

    • Use the Merge data tool to transfer all items which rely on your addition to the centrally supported equivalent. At the end of this process, your addition will be deleted.

    • You can now export the items associated with the centrally supported entry.

In the underlying database, this information is stored in field SYSTEM_SUPPLIED_DATA which is present in many tables. It is a boolean (Yes/No) field.
