Navigating in the Recording Card

The Recording Card is designed to allow rapid entry using the keyboard, so navigation and entry without having to use the mouse is important. Use the following keys to move around and edit.

<Tab>

Moves between controls in the recording card (the species list grid is treated as one control)

<Space>

In the grid, ticks and unticks the species in the current row

<Enter>

Completes editing of the current grid cell and moves to the next cell

left and right arrow

In the grid, moves between cells unless you are in the middle of editing a cell when it moves the insertion point a character to the left or right.

up and down arrow

In the grid, moves to the same cell in the previous or next row, unless you are selecting from a drop-down list, moves to the previous or next item in the list.

<PgUp> and <PgDn>

In the grid, moves up or down the list by a screen-full.

<F2>

Look-up. If a grid-cell contains a value looked up from elsewhere (like a Biotope) then <F2> will open the appropriate window to look up a value. Select the required value in the new window and press <F9> to return the data.

In a Spatial Ref column, this opens the Map window allowing a spatial reference to be obtained by clicking on the map.

To enter data by keyboard:

  • Move between prompts using <Tab>

  • In most fields, type a value and then press <Enter>

  • In the Species List grid, move around using the cursor keys.

  • Select and unselect species by positioning the cursor in the correct row and pressing <Space>

  • Fill in the grid cells by moving to the correct cell using the cursor keys and then type the value you want followed by <Enter>.

  • If the cell has a pop-up list of options, use the Up and Down arrow keys, or type the first letter of the option you want, to make a selection and then press <Enter>

  • If the cell requires you to lookup a value from another window, press <F2>, select the value in the new window and press <F9> to return it.

Tip: Whilst it is undoubtedly quicker to type in data using the keyboard only, there are still some operations where using the mouse can be faster. For example, if you look up a value (like a Location name or a species name) and are presented with a lengthy list of possible matches, it is usually quicker to double-click the one you want than it is to navigate to it using the cursor keys.
