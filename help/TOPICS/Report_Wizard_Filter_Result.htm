Report Wizard Filter Result

This screen lists your records in a simple grid.

From this screen you can choose what to do with the selected data. Click on the [Report output] button and the following options will appear;

 

 

General Options

  • Print Grid prints out the grid of data as it appears on the screen.

  • Copy copies just the selected cell to the Windows clipboard so that it can be pasted into a file such as an excel spreadsheet or word document.

  • Copy All copies all the data to the Windows clipboard so that it can be pasted into a file such as an excel spreadsheet or word document. Alternatively you can use Export to Excel to save the data in a new excel file.

  • Send to Map does a distribution map of the data - see Distribution maps.

  • Revalidate Items allows you to revalidate the items in the filter result - see Validating data

Note: the facility to revalidate items is only available to System Managers - see User access levels.

Export Options

  • Export Data - the function of this option is to export the observations listed in a form suitable for importing into another copy of Recorder. Note that in this case it is not just the data on view in the filter result that is exported, but all data associated with these observations. An export type needs to be selected and the types on offer will be the same as when doing other exports - see Exports. If the appropriate add-in has been installed, exports to other systems will also be possible, e.g. to DMAP or the NBN Gateway. When data is exported this way the Report Wizard is being used to select the observations for export; what is exported will be dependent on the export type not on the attributes selected when going through the wizard.

  • Export to Excel allows you to save the data on view in the filter result to an excel file.

  • Export to Google Earth allows you to save the data in a format suitable for use with Google Earth.

  • Export to ESRI Shape File allows you to save the data in a format suitable for use with GIS systems like ArcView. The next dialogue box allows you to select the attributes to be exported.

  • Export to Other Formats opens an Export Wizard that allows you to save the data in a number of other formats, e.g. a Text file (*.txt), a HTML file (*.htm) or a Lotus 1-2-3 file (*.wk1).

Report Templates

  • New Report Template will be on offer, i.e. not greyed out, if you selected New Layout in the Layout Selection window as you worked your way through the wizard. When selected the report designer opens so that you can design a report layout which will enable you to print the data in a format suitable for giving to customers - see Report designer for details.

  • Run/Edit Report Template will be on offer if you opted to use an existing report in the Layout Selection window. Existing report layouts/templates can be reused any number of times. They can also be edited and resaved, or edited and saved as a new layout, for future use.

Snapshots

Data may be exported in a simplified database structure for bespoke analysis or for use in another programme such as a GIS package. Snapshots may be saved and used again at a later time or edited for future use. For information about snapshots please see help file on Reporting-The snapshot tool

At this point you can use the [Back] button to go back through the screens of the Wizard and modify your selections. If you choose [Finish], the Wizard will terminate. Once you do this, you cannot go back and edit that report. You can also change the order of the columns by dragging and dropping the cell that contains the column heading. This order is remembered if the report is saved.

You can save a complete report (including the criteria from the Report Wizard and a report layout from the Report Designer). Once saved, it can be run again at any time (from Reports - Run on the main menu). When you run a previously saved report like this, the result reflects the current state of the database.

The green filter bar at the top allows you to filter the observations in the filter result. To apply a filter:

  • Type some filter text in the green box at the top of the appropriate column and press <Enter>. For example, if you typed "bell" <Enter> in the green box underneath Taxon Common Name, the system would return just the observations where the data in that column started with "bell". If you typed "%bell" <Enter> the system would return the observations where the data in that column contained "bell".

To remove a filter:

  • Delete the text in the green box at the top of the appropriate column and press <Enter>.

Another facility available from the Filter Results window is the ability to examine (and edit) the individual items shown in the grid. To do this:

  • Double-click in a cell in the grid.

OR

  • Select a cell by clicking in it with your mouse or moving the highlight bar around using the arrow keys.

  • Then click the button in the active toolbar.

The observation window will open, filtered to display the selected survey event, sample or observation. Use the [Show All] button (bottom right) to remove the filter. All data in the window will then be accessible again via the hierarchy, or list, in the left-hand panel, and the item that was selected when the [Show All] button was clicked will be highlighted.

Note: if you select a cell from a taxon occurrence (species observation), e.g. a Taxon Latin Name, then the Observations hierarchy opens with a filter on it so that the taxon occurrence is on view with its parent entries but not any other observation data.

Note: if you select a cell from a sample, e.g. a Sample Date, then the Observations hierarchy opens with a filter on it so that the sample is on view along with its parent entries and all the taxon occurrences in it but not any other observation data.

Once an observation is displayed in the Observations hierarchy, it can be edited as normal. A good way of locating and correcting errors is therefore to use the Wizard to generate a report that will find the incorrect data (e.g. use a bounding box to find observations with spatial references that are in the wrong area, or set an Additional Filter to select dates that are outside the expected range). Then access and edit the erroneous items from the Filter Result window.
