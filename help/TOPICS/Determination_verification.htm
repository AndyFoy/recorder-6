Verification of identifications

Taxon and biotope occurrence records have a Verified flag to indicate whether or not the taxon or biotope have been identified correctly. It can have one of three states:

0

Not verified

Verification has not been attempted. This is the default and all records will be flagged as Not verified unless the user specifies otherwise.

1

Failed/pending verification

The taxon name attributed to this record has been checked and is believed to be incorrect (a reason should preferably be given in the comment), or it is awaiting verification.

2

Passed verification

The taxon name attributed to this record has been checked and is believed to be correct.

The value of the Validation flag is shown on the General tab of the Taxon and Biotope occurrences.

How is the Verified flag set?

It is set automatically depending on the determination type on the preferred determination for the Taxon or Biotope occurrence. The values are:

Determination Type

Verified flag

Correct

2 - Passed verification

Considered Correct

2 - Passed verification

Considered Incorrect

1 - Failed/pending verification

Incorrect

1 - Failed/pending verification

Requires Confirmation

1 - Failed/pending verification

Unconfirmed

0 - Not verified

The determination types listed above were introduced in Recorder 6 version 6.13 in response to user's requests for improved verification flagging. They are system supplied entries in that version. The following determination types were present in earlier versions of the system as system supplied entries. They are also present in version 6.13 but if you have installed Recorder 6 using a version 6,13 installation CD you will find that they are flagged as hidden so that they can't be used for data entry.

Determination Type

Verified flag

Confirmation

2 - Passed verification

Validation

2 - Passed verification

Invalid

1 - Failed/pending verification

Observation

0 - Not verified

Original

0 - Not verified

Note: The determination type is a Term List which users can add to if they wish by selecting Tools - Term Lists from the main menu.

Note: If an observation is flagged as Failed/pending verification:

  • It will not be exported in certain circumstances - see the notes in red in Exports.

  • It will not appear in reports unless you specifically indicate that observations flagged as Failed/pending verification should be included (Reports – Wizard – Additional filters – Constraints).
