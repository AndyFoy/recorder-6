Group band report 3

Linked group bands

Within each location, we want the observations grouped by date. To do this we need to add another group band and make Sample Date its controlling field. But, in addition, we need to link it with the existing band controlled by location.

To see why this is necessary, consider the following observations sorted by location and date:

Date 1

Location 1

Date 1

Location 1

Location 2

Date 2

If we set up two group bands controlled by Location and Date we would see the following bands printed:

Location 1

1st Group band prints

Date 1

2nd group band prints

Date 2

2nd group band prints because date has changed

Location 2

1st Group band prints because location has changed

However, the 2nd group band won't print again, following Location 2, because the date hasn't changed.

We can get over this by linking the second group band to the first, What this does in effect is to "reset" the linked band to start again from scratch when its parent band is triggered. If the 2nd group band were linked to the first, we would see the result we want:

Location 1

1st Group band prints

Date 1

2nd group band prints

Date 2

2nd group band prints because date has changed

Location 2

1st Group band prints, because location has changed, and resets 2nd band

Date 2

2nd group band prints
