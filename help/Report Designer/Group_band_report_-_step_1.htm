Group band report

What are group bands?

So far we have seen a simple columnar report. It is possible to produce far more complex reports using Group Bands. A Group band has a special field (or expression) assigned to it and the band ONLY PRINTS WHEN THE VALUE OF THAT FIELD OR EXPRESSION CHANGES.

Example

Consider a small subset of data.

Aberithon Turbary station 1

Anasimyia lineata

17/07/1989

Dave Reid

Aberithon Turbary station 1

Helophilus pendulus

17/07/1989

Dave Reid

Aberithon Turbary station 1

Helophilus pendulus

10/10/1989

Dave Reid

Aberithon Turbary station 1

Neoascia tenur

17/07/1989

Dave Reid

As all observations are from the same sampling station, there is no need to keep repeating the locality. It would be better to lay this out so that the locality appears just once at the top.

Aberithon Turbary station 1

Group band

Detail

Anasimyia lineata

17/07/1989

Dave Reid

Detail

Helophilus pendulus

17/07/1989

Dave Reid

Detail

Helophilus pendulus

10/10/1989

Dave Reid

Detail

Neoascia tenur

17/07/1989

Dave Reid

By putting the location name on a Group band, which is only printed when the locality changes, we could achieve this. Note that we still have a detail band, containing the species name, date and the determiner's name, which prints for each row of data.

Similarly you may wish to group the information by sampling date for that sampling station like this:

Aberithon Turbary station 1

17/07/1989

Anasimyia lineata

Dave Reid

Helophilus pendulus

Dave Reid

Neoascia tenur

Dave Reid

10/10/1989

Helophilus pendulus

Dave Reid

This needs two Group bands, the first to print when the location changes and the second when the date changes. In both cases. Now, the detail band only contains species name and the determiner's name.

Group bands can be shown either as "Group headers" which print a sub-heading at the beginning of a group (as above) or "Group footers. Footers are particularly useful for printing sub-totals and other statistical data. Groups can be nested to an unlimited number of levels, making very complex layouts possible.

Note that the way in which the report is sorted is absolutely critical when you use group bands. To work properly in this case the rows MUST be sorted by location and then by date. Group bands print whenever their controlling expression changes, so if the rows are not sorted so as to group rows correctly, the report will not print as intended.
